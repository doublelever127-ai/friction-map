# 마찰지도 Google Play Data Safety 작성 메모

작성일: 2026-04-29  
대상 버전: v0.3.0 Android closed test

이 문서는 Play Console의 Data Safety 항목을 작성할 때 참고하기 위한 내부 메모입니다. 실제 제출 전 Play Console 화면의 최신 질문 문구에 맞춰 다시 확인합니다.

## 1. 현재 앱 구조

- 서버 없음
- 로그인 없음
- 데이터베이스 없음
- 광고 SDK 없음
- 분석 SDK 없음
- AI API 없음
- 기록은 기기 안에 저장
- 사용자가 기록 내보내기 가능
- 사용자가 모든 기록 삭제 가능

## 2. 사용자 데이터 처리 방향

첫 출시 버전에서 앱 개발자는 사용자의 기록을 서버로 수집하지 않습니다.

앱 안에서 사용자가 직접 입력한 기록은 앱 기능을 위해 기기 안에 저장됩니다.

저장되는 기록 예:

- 막힌 순간 한 줄 기록
- 그때 느낌
- 어느 쪽 일이었는지
- 어디에서 막혔는지
- 얼마나 버거웠는지
- 작게 바꿔보기 카드 내용
- 생성/수정 시각

## 3. 수집 여부 판단 메모

Google Play의 Data Safety 문항에서 `수집`은 앱 밖으로 전송되는지 여부와 관련해 해석될 수 있으므로, 실제 제출 전 Play Console의 설명을 기준으로 판단합니다.

현재 설계 기준:

- 서버로 보내지 않음
- 제3자에게 공유하지 않음
- 광고나 분석 목적으로 사용하지 않음
- 앱 기능 표시를 위해 기기 안에 저장

## 4. 공유 여부

현재 버전에서는 사용자 기록을 제3자와 공유하지 않습니다.

## 5. 보안 전송

현재 버전에서는 사용자 기록을 서버로 전송하지 않습니다.

향후 서버 백업, 로그인, 분석 SDK, AI API를 추가하면 보안 전송, 암호화, 제3자 공유 여부를 다시 작성해야 합니다.

## 6. 삭제 요청

앱 안에서 사용자가 직접 모든 기록을 삭제할 수 있습니다.

앱에는 `내 기록 관리` 영역이 있으며, 여기서 아래를 수행합니다.

- 기록 내보내기
- 모든 기록 삭제

## 7. 개인정보처리방침 URL 준비

Play Console 제출 전 공개 접근 가능한 개인정보처리방침 URL이 필요합니다.

후보:

```text
https://doublelever127-ai.github.io/friction-map/privacy-policy/
```

또는 별도 공개 페이지:

```text
https://friction-map.example.com/privacy
```

GitHub 저장소가 private이면 GitHub 문서 링크는 개인정보처리방침 URL로 적합하지 않습니다. 공개 접근 가능한 웹 URL을 준비해야 합니다.

## 8. 출시 전 다시 확인할 항목

- Play Console Data Safety 질문의 최신 문구
- 개인정보처리방침 URL 공개 접근 가능 여부
- 앱 안 개인정보 안내와 정책 문서 내용 일치 여부
- 실제 SDK 목록
- 광고/분석/로그인/AI API 미사용 여부
- Android 권한 목록

## 9. 현재 Android 권한

현재 Android Manifest에는 인터넷 권한이 포함되어 있습니다.

```text
android.permission.INTERNET
```

Capacitor WebView 앱에서 기본적으로 포함될 수 있으나, 현재 앱 기능은 서버 전송을 사용하지 않습니다. Play Console 제출 전 실제 네트워크 호출이 없는지 QA에서 다시 확인합니다.
