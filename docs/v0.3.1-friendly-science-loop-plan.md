# 마찰지도 v0.3.1 친숙한 용어 + 과학 기반 행동 루프 재설계 계획

작성일: 2026-04-30  
목표: 현재의 `남기기 → 지도 → 시도 → 내 기록` 구조를 `남기기 → 지도 → 시도 → 돌아보기`의 행동 자기조절 루프로 재설계한다.

## 1. 왜 기존 구조를 바꿔야 하는가

v0.3.1의 하단 탭 구조는 긴 스크롤 화면을 모바일 앱처럼 나누는 데 성공했습니다. 다만 실사용자 관점에서는 마지막 탭인 `내 기록`이 “내가 다음에 무엇을 하면 되는지”보다 “데이터를 관리하는 곳”으로 느껴질 수 있습니다.

마찰지도의 핵심 가치는 단순히 기록을 쌓는 것이 아니라, 다음 흐름을 반복하게 돕는 데 있습니다.

```text
막힌 순간을 남긴다
→ 자주 막히는 위치를 본다
→ 다음에 덜 버겁게 해볼 작은 방법을 정한다
→ 해보니 어땠는지 돌아보고 더 작게 조정한다
```

따라서 마지막 화면은 `관리`나 `내 기록`보다 `돌아보기`가 더 적합합니다. 기록 관리, 내보내기, 삭제, 개인정보 안내는 여전히 필요하지만, 사용자의 핵심 행동 루프를 끊지 않도록 `돌아보기` 화면 하단에 배치합니다.

## 2. 기존 구조의 한계

현재 구조:

```text
남기기 | 지도 | 시도 | 내 기록
```

장점:

- 첫 화면이 기록 중심으로 단순해졌습니다.
- 지도와 시도를 별도 화면으로 나누어 모바일 사용성이 좋아졌습니다.
- 기록 관리와 개인정보 안내가 한곳에 모였습니다.

한계:

- `내 기록`은 행동 루프의 끝이라기보다 보관함처럼 느껴질 수 있습니다.
- 시도 카드를 만든 뒤 “그 다음 무엇을 해야 하는지”가 약합니다.
- 상태 변경은 가능하지만, 사용자가 경험을 짧게 돌아보고 조정하는 흐름은 아직 부족합니다.
- “작게 시도하기”가 한 번의 작성으로 끝나고, 다음 시도로 이어지는 느낌이 약합니다.

## 3. 새 행동 루프

새 구조:

```text
남기기 | 지도 | 시도 | 돌아보기
```

행동 루프:

```text
남기기
오늘 막힌 순간을 한 줄로 남긴다.

지도
어떤 일에서, 어떤 순간에 자주 막히는지 본다.

시도
다음에 덜 버겁게 해볼 작은 방법을 정한다.

돌아보기
해보니 어땠는지 살펴보고, 다음 시도를 더 작게 조정한다.
```

이 루프의 목적은 사용자를 평가하는 것이 아니라, 사용자가 자신의 행동 맥락을 더 잘 관찰하고 다음 행동을 더 작게 만들 수 있게 돕는 것입니다.

## 4. 각 화면의 사용자 목적

| 화면 | 사용자 목적 | 화면에서 보이는 말 |
| --- | --- | --- |
| 남기기 | 오늘 막힌 순간을 한 줄로 남긴다 | “오늘 어디서 막혔나요?” |
| 지도 | 자주 막히는 위치를 본다 | “자주 막힌 위치” |
| 시도 | 다음에 덜 버겁게 해볼 작은 방법을 정한다 | “작게 바꿔보기” |
| 돌아보기 | 해보니 어땠는지 보고 다음 시도를 조정한다 | “돌아보기” |

## 5. 각 화면의 과학적 기능

과학 용어는 앱 화면에 직접 노출하지 않습니다. 아래 개념은 내부 설계 기준입니다.

| 화면 | 내부 과학적 기능 | 사용자에게 보이는 경험 |
| --- | --- | --- |
| 남기기 | 자기관찰, 감정 라벨링 | “막힌 순간을 한 줄로 남기기” |
| 지도 | 패턴 피드백, 행동 맥락 피드백 | “자주 막힌 위치 보기” |
| 시도 | 실행의도, 작은 행동 설계, 실행 가능성 확인 | “다음에 덜 버겁게 해볼 방법 정하기” |
| 돌아보기 | 회고, 피드백 루프, 문제해결, 조정 | “해보니 어땠는지 살펴보고 더 작게 바꾸기” |

## 6. 사용자용 용어 사전

| 내부/기획 용어 | 화면 용어 | 사용 이유 |
| --- | --- | --- |
| 마찰 | 막힘, 막힌 순간 | 처음 사용자에게 더 직관적입니다. |
| 자기관찰 | 남기기 | 과학 용어를 행동 문구로 바꿉니다. |
| 피드백 | 지도에서 보기 | 분석보다 확인에 가깝게 느껴집니다. |
| 실행의도 | 언제/어디서/무엇을 해볼까요? | 사용자가 바로 적을 수 있는 질문입니다. |
| 작은 행동 설계 | 작게 바꿔보기 | 부담을 줄입니다. |
| 실행 가능성 | 이 정도면 해볼 만한가요? | 사용자가 스스로 크기를 조절하게 합니다. |
| 회고 | 돌아보기 | 일상적인 말입니다. |
| 실패 | 잘 안 됐을 때 | 판정이 아니라 관찰로 보이게 합니다. |
| 상태 | 지금 어디쯤인가요? | 평가보다 위치 표시처럼 느껴집니다. |
| 보류 | 잠시 멈춤 | 부정적 느낌이 덜합니다. |

피해야 할 표현:

- 실패
- 의지 부족
- 문제 영역
- 진단
- 치료
- 개선 효과 보장
- 정신건강 관리
- 심리 처방

## 7. 시도 카드 데이터 구조 개선안

현재 `FrictionExperiment`는 다음 흐름을 담고 있습니다.

```ts
FrictionExperiment:
- id
- frictionLogId
- title
- hypothesis
- action
- durationDays
- successCriteria
- failureInterpretation
- status
- createdAt
- updatedAt
```

v0.3.1 이후에는 `시도`와 `돌아보기` 루프를 위해 아래 필드를 검토합니다.

```ts
FrictionAttempt:
- id
- frictionLogId
- title
- whyBlocked
- ifSituation
- thenAction
- durationDays
- checkMethod
- fallbackPlan
- feasibilityRating
- status
- review
- createdAt
- updatedAt
- reviewedAt
```

필드 의미:

| 필드 | 사용자 질문 | 내부 기능 |
| --- | --- | --- |
| whyBlocked | 왜 막혔을까요? | 원인 가설 |
| ifSituation | 언제/어디서 다시 막힐 것 같나요? | 실행의도 조건 |
| thenAction | 그때 무엇을 작게 해볼까요? | 실행의도 행동 |
| checkMethod | 어떻게 확인할까요? | 관찰 기준 |
| fallbackPlan | 잘 안 되면 어떻게 더 작게 줄일까요? | 조정 계획 |
| feasibilityRating | 이 정도면 해볼 만한가요? | 실행 가능성 확인 |
| review | 해보니 어땠나요? | 회고 |

## 8. 실행의도 기반 입력 구조

현재 시도 카드는 “왜 막혔는지”와 “무엇을 해볼지”를 적게 합니다. 다음 단계에서는 사용자가 상황과 행동을 더 구체적으로 연결하도록 합니다.

화면 문구 예:

```text
언제 다시 막힐 것 같나요?
예: 퇴근 후 운동복을 입기 전

그때 무엇을 작게 해볼까요?
예: 운동을 시작하지 않고 운동복만 입어본다
```

내부 구조:

```text
만약 [상황]이면, [작은 행동]을 해본다.
```

화면에는 “실행의도”라는 말은 쓰지 않습니다.

## 9. “이 정도면 해볼 만한가요?” 기능

작은 행동이 실제로 작아졌는지 확인하기 위한 입력입니다.

화면 문구:

```text
이 정도면 해볼 만한가요?
```

선택지 예:

```text
아직 큼 / 조금 버거움 / 해볼 만함 / 꽤 가벼움
```

권장 동작:

- 사용자가 `아직 큼` 또는 `조금 버거움`을 고르면 행동을 더 작게 줄이라는 안내를 보여줍니다.
- 저장을 막지는 않습니다.
- 앱이 정답을 추천하지 않고, 사용자가 스스로 조정하게 합니다.

문구 예:

```text
조금 더 작게 줄여도 괜찮아요.
```

## 10. 돌아보기 기능 구조

`돌아보기` 화면은 시도 카드의 결과를 성공/실패로 판정하는 곳이 아닙니다. 사용자가 “해보니 어땠는지”를 보고 다음 시도를 조정하는 곳입니다.

포함할 요소:

- 현재 시도 카드 목록
- 상태 변경
- 짧은 돌아보기 입력
- 다음에 더 작게 해볼 것
- 내 기록 관리 섹션
- 개인정보 안내

돌아보기 질문:

```text
해보니 어땠나요?
```

선택지 예:

```text
생각보다 가벼웠어요
조금 버거웠어요
상황이 달랐어요
아직 해보지 못했어요
```

후속 질문:

```text
다음에는 어떻게 더 작게 해볼까요?
```

예시:

```text
운동복을 입는 것도 크다면, 운동복을 꺼내두기만 해본다.
```

## 11. 기존 실험 카드 데이터 마이그레이션 방향

기존 데이터는 삭제하지 않고 유지합니다.

현재 필드와 새 필드의 대응:

| 기존 필드 | 새 필드 후보 | 처리 |
| --- | --- | --- |
| hypothesis | whyBlocked | 그대로 이동 가능 |
| action | thenAction | 그대로 이동 가능 |
| durationDays | durationDays | 유지 |
| successCriteria | checkMethod | 이름만 바꿈 |
| failureInterpretation | fallbackPlan | 더 부드러운 의미로 재해석 |
| status | status | 내부 값 유지, 화면 표시만 변경 |
| createdAt | createdAt | 유지 |
| updatedAt | updatedAt | 유지 |

마이그레이션 원칙:

- 기존 localStorage 키를 바로 바꾸지 않습니다.
- 새 필드가 없으면 화면에서 기존 필드를 읽어 표시합니다.
- 새 구조가 안정화되면 `friction-map.attempts.v1` 같은 새 키를 검토합니다.
- 사용자가 만든 기존 시도 카드는 깨지면 안 됩니다.

## 12. 내 기록/설정 배치

기존 `내 기록` 화면의 기능은 사라지지 않습니다. 다만 하단 탭의 이름은 `돌아보기`로 바꾸고, 기록 관리는 화면 하단에 둡니다.

돌아보기 화면 구성:

```text
[돌아보기]
해보니 어땠는지 살펴보고, 다음 시도를 더 작게 조정해보세요.

[시도 카드 돌아보기]
- 상태
- 짧은 회고
- 다음에 더 작게 해볼 것

[내 기록 관리]
- 전체 기록 보기
- 기록 내보내기
- 모든 기록 삭제

[개인정보 안내]
- 기록은 이 기기 안에 저장됩니다.
- 서버로 전송하지 않습니다.
- 로그인 없이 사용할 수 있습니다.
```

전체 기록 목록이 너무 길면 기본은 최근 5개만 보여주고, `전체 보기`로 확장합니다.

## 13. Play Store 문구 주의사항

마찰지도는 치료, 진단, 의료, 정신건강 개선 앱으로 표현하지 않습니다.

권장 표현:

```text
자꾸 막히는 순간을 한 줄로 남기고, 반복되는 패턴을 보는 개인 기록 앱
```

사용 가능한 표현:

- 개인 기록 앱
- 생활 패턴 기록
- 막힌 순간 기록
- 자주 막힌 위치
- 다음에 덜 버겁게 해볼 작은 방법
- 로컬 저장
- 로그인 없이 사용

피해야 할 표현:

- 불안 치료
- 우울 개선
- 심리 진단
- 정신건강 관리
- 치료 효과
- 행동 교정
- 습관 문제 해결 보장

## 14. 구현 순서

1. 문서 기준 확정
2. 하단 탭을 `남기기 | 지도 | 시도 | 돌아보기`로 변경
3. `돌아보기` 화면을 먼저 빈 상태/목록 중심으로 추가
4. 기존 `내 기록` 기능을 `돌아보기` 하단으로 이동
5. `ExperimentList`에 짧은 돌아보기 영역 추가
6. 시도 카드 입력에 `언제/어디서 다시 막힐 것 같나요?` 질문 추가
7. 시도 카드 입력에 `이 정도면 해볼 만한가요?` 선택 추가
8. 기존 실험 카드 데이터와 새 필드 호환 처리
9. localStorage 마이그레이션 보완
10. 모바일 QA
11. `pnpm.cmd build`
12. Android AAB 재생성

## 15. QA 체크리스트

기능:

- [ ] 남기기 화면에서 기록을 추가할 수 있다.
- [ ] 저장 후 지도에 개수가 반영된다.
- [ ] 지도 화면에서 Top 3가 정확하다.
- [ ] 시도 카드를 만들 수 있다.
- [ ] 기존 시도 카드가 깨지지 않는다.
- [ ] 돌아보기 화면에서 시도 카드 상태를 볼 수 있다.
- [ ] 돌아보기 입력이 저장된다.
- [ ] 기존 기록 내보내기와 전체 삭제가 유지된다.
- [ ] 새로고침 후 기록과 시도 카드가 유지된다.

문구:

- [ ] 화면에 “자기관찰”, “실행의도”, “COM-B”, “행동변화기법” 같은 용어가 직접 보이지 않는다.
- [ ] “실패”, “의지 부족”, “문제 영역” 같은 표현이 불필요하게 보이지 않는다.
- [ ] 치료나 진단처럼 보이는 표현이 없다.
- [ ] 사용자는 `남기기 → 지도 → 시도 → 돌아보기` 흐름을 이해할 수 있다.

모바일:

- [ ] 하단 탭 4개가 한 화면에 안정적으로 보인다.
- [ ] 돌아보기 화면이 너무 긴 설정 화면처럼 보이지 않는다.
- [ ] 기록 관리 섹션은 하단에 자연스럽게 배치된다.
- [ ] 시도 카드 입력은 손가락으로 누르기 쉽다.
- [ ] 지도는 가로 스크롤이 유지된다.

| 지도 확인율 | 기록 후 지도 화면을 본 사용자 / 기록한 사용자 | 피드백 루프 진입 |
| 시도 카드 생성률 | 시도 카드 1개 이상 만든 사용자 / 기록 1개 이상 사용자 | 작은 행동 설계 진입 |
| 돌아보기 작성률 | 돌아보기 1개 이상 작성한 사용자 / 시도 카드 생성 사용자 | 자기조절 루프 완성 |
| 재기록률 | 2일 이상 기록한 사용자 / 전체 사용자 | 반복 사용 가능성 |
| 기록-시도 전환율 | 시도 카드 생성 수 / 기록 수 | 패턴에서 행동으로 이어진 정도 |
| 삭제/내보내기 발견률 | 해당 버튼을 확인하거나 사용한 사용자 / 전체 사용자 | 데이터 통제 인식 |

정성 질문:

```text
1. 처음 앱을 열었을 때 무엇을 하는 앱인지 이해됐나요?
2. “막힌 순간”이라는 말이 이해됐나요?
3. 지도 화면에서 무엇을 봐야 하는지 알 수 있었나요?
4. 시도 카드는 부담스럽지 않았나요?
5. 돌아보기 화면은 다음 시도를 정하는 데 도움이 됐나요?
6. 치료나 진단 앱처럼 느껴지지는 않았나요?
7. 기록이 이 기기 안에 저장된다는 점이 신뢰감을 줬나요?
```

성공 기준 초안:

- 12명 중 9명 이상이 첫 기록을 남긴다.
- 12명 중 7명 이상이 지도 화면을 확인한다.
- 기록한 사용자 중 40% 이상이 시도 카드를 만든다.
- 시도 카드를 만든 사용자 중 50% 이상이 돌아보기를 1회 이상 남긴다.
- 사용자 인터뷰에서 치료/진단 앱처럼 느꼈다는 응답이 나오지 않는다.

## 17. 중요한 한계

이 설계는 행동 자기조절을 돕기 위한 개인 기록 루프입니다. 효과를 보장하지 않습니다.

말하면 안 되는 것:

- “마찰지도는 미루는 습관을 고칩니다.”
- “마찰지도는 불안을 줄여줍니다.”
- “마찰지도는 심리 문제를 진단합니다.”
- “마찰지도는 행동 개선 효과가 검증되었습니다.”

말할 수 있는 것:

- “막힌 순간을 한 줄로 남기고 반복되는 패턴을 볼 수 있습니다.”
- “다음에 덜 버겁게 해볼 작은 방법을 직접 정할 수 있습니다.”
- “기록은 기기 안에 저장되며, 사용자가 삭제하거나 내보낼 수 있습니다.”
