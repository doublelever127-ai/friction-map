# 마찰지도 v0.3.2 감성적 프리미엄 UI 리디자인 계획

## 1. 현재 UI의 한계

v0.3.1까지의 마찰지도는 기능 흐름이 명확해졌다. 사용자는 하단 탭을 통해 `남기기 -> 지도 -> 시도 -> 돌아보기` 흐름을 따라갈 수 있고, 각 화면의 목적도 한 페이지 대시보드보다 분명해졌다.

다만 현재 UI는 아직 "잘 정리된 기능 화면"에 가깝다. Play Store에 올라가는 실제 모바일 앱으로 보이기 위해서는 다음 한계를 줄여야 한다.

- 화면의 정보 구조는 좋아졌지만, 첫 인상이 아직 웹앱 또는 내부 도구처럼 느껴질 수 있다.
- 카드와 섹션의 스타일이 기능별로 분리되어 있지만, 하나의 브랜드 경험으로 묶이는 감성은 약하다.
- 입력, 지도, 시도, 돌아보기의 감정 온도가 비슷해서 화면마다 다른 목적이 충분히 느껴지지 않을 수 있다.
- 한국어 문구는 친숙해졌지만, 타이포 위계와 여백이 더 정돈되어야 프리미엄 앱처럼 보인다.
- 마찰 지도는 기능적으로 유용하지만, "내가 자주 막히는 위치를 조용히 비춰주는 지도"라는 감각을 더 강화할 수 있다.
- Bottom nav는 앱다운 구조를 만들었지만, 현재보다 더 가볍고 안정적인 네이티브 앱 느낌이 필요하다.
- 다크모드는 기능적으로 유지되지만, 단순 반전이 아니라 밤에 쓰기 편한 차분한 대비가 필요하다.

## 2. 목표 UI 인상

v0.3.2의 목표는 마찰지도를 기능 중심 웹 대시보드에서 감성적이고 세련된 모바일 앱 UI로 끌어올리는 것이다.

사용자가 앱을 열었을 때 느껴야 하는 첫 인상은 다음과 같다.

- 조용히 나를 맞아주는 앱
- 오늘 막혔던 순간을 편하게 남길 수 있는 공간
- 복잡한 분석보다 부드러운 관찰을 돕는 도구
- 생산성 앱처럼 명료하지만, 감정 기록 앱처럼 따뜻한 화면
- 치료나 진단이 아니라 개인 기록과 자기이해를 위한 앱

핵심 문장은 다음 감각으로 유지한다.

> 할 일을 더 늘리지 않습니다. 자주 막히는 순간을 찾아, 다음에 덜 버겁게 만듭니다.

## 3. 디자인 컨셉 키워드

- Soft Focus: 사용자의 감정에 과하게 개입하지 않고 부드럽게 초점을 잡는다.
- Quiet Premium: 눈에 띄는 장식보다 여백, 색감, 타이포, 질감으로 고급감을 만든다.
- Warm Clarity: 따뜻하지만 흐릿하지 않고, 해야 할 행동은 분명하게 보인다.
- Local-first Calm: 기록이 내 기기 안에 머문다는 안정감을 시각적으로 표현한다.
- Gentle Momentum: 기록에서 지도, 지도에서 시도, 시도에서 돌아보기로 부담 없이 이어진다.

## 4. 컬러 방향

컬러는 자극적인 생산성 팔레트보다 따뜻한 중립색과 낮은 채도의 포인트 색을 사용한다.

### 기본 라이트 팔레트

- 배경: 따뜻한 아이보리 계열 `#F8F5EF`, `#F4F0E8`
- 표면: 부드러운 흰색 `#FFFFFF`, 크림 화이트 `#FFFDF8`
- 기본 텍스트: 깊은 차콜 `#26231F`
- 보조 텍스트: 웜 그레이 `#746D64`
- 라인: 옅은 샌드 그레이 `#E4DDD2`
- 주요 포인트: 세이지 그린 `#6F8F7A`
- 보조 포인트: 더스티 코랄 `#C98273`
- 지도 강조: 세이지에서 딥 올리브로 이어지는 단계형 색상

### 다크 팔레트

- 배경: 푸른빛이 적은 웜 블랙 `#171512`
- 표면: 다크 브라운 차콜 `#211E1A`
- 강조 표면: `#2A261F`
- 기본 텍스트: 웜 아이보리 `#F4EDE2`
- 보조 텍스트: `#B8AEA1`
- 라인: `#3B352D`
- 주요 포인트: 차분한 세이지 `#9DB58E`
- 보조 포인트: 낮은 채도 코랄 `#D59A8D`

주의할 점:

- 화면 전체가 베이지 한 톤으로만 읽히면 안 된다.
- 포인트 컬러는 버튼, 선택 상태, 지도 강도에 제한적으로 사용한다.
- 경고처럼 보이는 빨강은 삭제 확인 같은 명확한 상황에서만 낮은 채도로 사용한다.
- 치료/명상 앱처럼 과하게 몽환적인 보라색 그라데이션은 피한다.

## 5. 타이포 방향

한국어 UI는 글자 크기보다 줄간격과 위계가 중요하다.

- 앱 제목: 22~24px, 굵기 700, 자간 0
- 화면 제목: 24~28px, 굵기 700, 짧고 명확하게
- 섹션 제목: 17~19px, 굵기 700
- 본문 설명: 14~15px, 줄간격 1.55~1.7
- 보조 정보: 12~13px, 줄간격 1.45
- 칩/버튼: 14px, 굵기 600

원칙:

- 모바일 첫 화면에서 큰 제목은 한 줄 또는 두 줄 안에 들어와야 한다.
- 설명문은 길게 쓰지 않고 2문장 이하로 유지한다.
- "마찰", "가설", "실험" 같은 내부 개념은 필요한 곳에서만 쓰고, 첫 사용 화면은 "막힘", "작게 시도하기", "돌아보기"를 우선한다.
- 자간은 0으로 유지한다.

## 6. Spacing / Radius / Shadow 시스템

### Spacing

- 화면 좌우 여백: 모바일 20px, 작은 기기 16px
- 섹션 간격: 28~36px
- 카드 내부 여백: 18~22px
- 입력 그룹 간격: 18~24px
- 칩 간격: 8px
- Bottom nav와 콘텐츠 하단 간격: 최소 88px

### Radius

- 앱 프레임: 28px 이상
- 주요 카드: 20~24px
- 작은 카드: 16~18px
- 칩: 999px
- 버튼: 14~16px
- 입력창: 18~20px

### Shadow

그림자는 "띄워 보이기"보다 "종이와 표면의 층"을 만드는 수준으로 쓴다.

- 기본 카드: 매우 약한 그림자 + 얇은 border
- 선택 카드: border 색상과 배경 톤으로 강조
- 다크모드: shadow보다 border와 표면 밝기 차이로 계층을 표현

예시:

- 라이트 카드: `shadow-[0_14px_40px_rgba(54,45,35,0.06)]`
- 다크 카드: `shadow-none`, border `rgba(255,255,255,0.08)`

## 7. 버튼 스타일

버튼은 조용하지만 눌러야 할 행동이 분명해야 한다.

### Primary

용도:

- 기록하기
- 시도 카드 만들기
- 내보내기 같은 긍정적 주요 행동

스타일:

- 배경: 세이지 그린 또는 깊은 웜 차콜
- 텍스트: 아이보리 또는 흰색
- 높이: 최소 48px
- radius: 16px
- 모바일 전체 폭 사용 가능

문구 예:

- 기록하기
- 시도 카드 만들기
- 기록 내보내기

### Secondary

용도:

- 지도에서 보기
- 남기러 가기
- 이전 화면으로 돌아가기

스타일:

- 배경: 낮은 채도 표면색
- border: 옅은 라인
- 텍스트: 기본 텍스트 또는 포인트 텍스트

### Ghost

용도:

- 삭제
- 접기/펼치기
- 보조 이동

스타일:

- 배경 없음
- hover/focus에서만 표면색 표시
- 삭제는 붉은색을 크게 쓰지 않고 텍스트 톤만 낮은 채도 코랄로 표현

## 8. 칩 스타일

칩은 v0.3.2에서 가장 중요한 상호작용 요소 중 하나다. 사용자가 분류 양식을 작성하는 느낌이 아니라 가까운 느낌을 가볍게 고르는 경험이어야 한다.

스타일:

- 기본: 따뜻한 표면색, 얇은 border, 40px 이상 터치 높이
- 선택됨: 배경 세이지 틴트, border 세이지, 텍스트 진한 차콜
- 다크 선택됨: 세이지가 너무 밝지 않게 낮은 채도 사용
- 칩 텍스트는 한 줄 유지, 긴 텍스트는 적절히 줄바꿈 또는 넓은 칩 허용

칩 그룹:

- 모바일에서 자연스럽게 wrap
- 줄 사이 간격 충분히 확보
- 선택 상태가 색상만으로 구분되지 않도록 border 또는 작은 점 표시 가능

## 9. 카드 스타일

카드는 "정보를 담는 상자"가 아니라 "조용한 단계"처럼 보여야 한다.

### 주요 카드

용도:

- 남기기 입력 카드
- 자주 막힌 위치 카드
- 작게 바꿔보기 작성 카드
- 돌아보기 카드

스타일:

- 둥근 모서리 22~24px
- 따뜻한 표면색
- 얇은 border
- 부드러운 shadow
- 상단에 짧은 제목과 한 줄 설명

### 보조 카드

용도:

- 최근 기록
- 상태 배지 영역
- 개인정보 안내
- 앱 정보

스타일:

- radius 16~18px
- shadow 거의 없음
- border 중심

### 카드 안의 카드 금지

중첩 카드가 많아지면 모바일에서 복잡해 보인다. 필요한 경우 카드 안에서는 구분선, 작은 배지, 배경 틴트 블록만 사용한다.

## 10. 입력창 스타일

입력창은 "양식"보다 "조용한 메모 공간"처럼 보여야 한다.

스타일:

- 높이: 기본 112~140px
- 배경: 카드보다 아주 살짝 다른 표면색
- border: 기본 옅게, focus에서 세이지 또는 차분한 포인트
- radius: 18~20px
- placeholder: 실제 사용 예시 중심

입력 문구:

- "예: 운동하려고 했는데 또 미뤘다"
- "예: 시작이 너무 커서 부담스러웠던 것 같다"
- "예: 운동을 목표로 하지 않고 운동복만 입어본다"

오류/안내:

- 빨간 오류 대신 부드러운 안내 톤
- 예: "한 줄만 남겨도 충분합니다. 막혔던 순간을 적어주세요."

## 11. Bottom Nav 스타일

하단 탭은 앱의 골격이다. 너무 도구 메뉴처럼 보이지 않고, 네이티브 앱처럼 안정적으로 보여야 한다.

탭:

- 남기기
- 지도
- 시도
- 돌아보기

스타일:

- 모바일 하단 고정
- 배경: 반투명 표면 + blur 또는 단단한 표면색
- 높이: 70~78px
- safe area 대응
- 활성 탭: 작은 pill 배경 또는 위쪽 점 표시
- 비활성 탭: 낮은 대비의 보조 텍스트
- 아이콘은 가능하면 lucide 아이콘 사용

주의:

- 탭 이름은 짧게 유지한다.
- "관리" 대신 "돌아보기"를 사용한다.
- 돌아보기 화면 하단에 기록 관리와 개인정보 안내를 배치한다.

## 12. 화면별 UI 리디자인 방향

### 12.1 남기기

목적:

오늘 막힌 순간을 한 줄로 남긴다.

첫 화면 인상:

- 화면을 열자마자 "여기엔 한 줄만 쓰면 된다"는 느낌이 들어야 한다.
- 지도, 요약, 설정이 끼어들지 않는다.

구성:

- 부드러운 인사형 제목: "오늘 어디서 막혔나요?"
- 짧은 설명: "미뤘던 일, 부담스러운 답장, 시작하지 못한 순간을 한 줄만 남겨보세요."
- 큰 메모형 textarea
- 예시 칩
- 그때 느낌 / 어느 쪽 일 / 어디에서 막혔는지 / 얼마나 버거웠는지
- 기록하기 버튼
- 최근 기록 1~2개

리디자인 포인트:

- 입력 카드가 화면의 70% 이상 주목을 가져야 한다.
- 빠른 예시 칩은 "예시로 채우기"처럼 보이되 실제 저장과 혼동되지 않게 한다.
- 저장 후 피드백은 작은 토스트 또는 카드 상단 메시지로 표시한다.

### 12.2 지도

목적:

어떤 일에서, 어떤 순간에 자주 막히는지 본다.

구성:

- 화면 제목: "자주 막힌 위치"
- 설명: "어떤 일에서, 어떤 순간에 자주 막히는지 지도처럼 볼 수 있어요."
- Top 3 카드
- 마찰 지도 히트맵
- 범례
- 반복 요약

리디자인 포인트:

- Top 3를 가장 먼저 보여준다.
- 히트맵은 표보다 타일 그리드처럼 보이게 한다.
- 숫자는 크고 명확하게, 보조 텍스트는 작게 둔다.
- 기록이 적을 때는 "아직 관찰을 시작한 단계"라는 부드러운 빈 상태를 둔다.

### 12.3 시도

목적:

다음에 덜 버겁게 해볼 작은 방법을 정한다.

구성:

- 화면 제목: "작게 바꿔보기"
- 설명: "바로 해결하지 않아도 괜찮아요. 다음에 덜 버겁게 해볼 작은 방법을 정해보세요."
- 기록 선택
- 왜 막혔을까요?
- 다음에는 어떻게 작게 해볼까요?
- 어떻게 확인할까요?
- 이 정도면 해볼 만한가요? 확인
- 시도 카드 만들기

리디자인 포인트:

- 긴 양식처럼 보이지 않게 단계형 블록으로 나눈다.
- 각 단계는 1, 2, 3 숫자보다 부드러운 라벨을 사용한다.
- "정답"이 아니라 "작게 시도"라는 느낌을 유지한다.
- 생성된 카드는 목록보다 "내가 정한 다음 시도"처럼 읽혀야 한다.

### 12.4 돌아보기

목적:

해보니 어땠는지 살펴보고, 다음 시도를 더 작게 조정한다.

구성:

- 화면 제목: "돌아보기"
- 설명: "해본 것을 평가하지 않고, 다음에 덜 버겁게 만들 단서를 살펴봅니다."
- 시도 카드 목록
- 상태 변경
- 돌아보기 메모
- 최근 기록 또는 전체 기록 진입
- 기록 내보내기
- 모든 기록 삭제
- 개인정보 안내
- 앱 정보

리디자인 포인트:

- 기록 관리 화면이 아니라 "내 기록을 챙기는 공간"처럼 보이게 한다.
- 삭제와 내보내기는 화면 아래쪽의 조용한 설정 영역에 둔다.
- 개인정보 안내는 신뢰감을 주되 법률 문서처럼 딱딱하지 않게 쓴다.

## 13. 마찰 지도 시각화 리디자인 방향

마찰 지도는 앱의 차별점이다. 표가 아니라 "반복되는 막힘의 위치를 비추는 지도"처럼 보여야 한다.

### 구조

- 상단: 자주 막힌 위치 Top 3
- 중간: 히트맵 타일
- 하단: 범례와 짧은 읽는 법

### 타일 스타일

- 0회: 거의 비어 있는 표면, 얇은 border
- 1회: 아주 연한 세이지 틴트
- 2~3회: 중간 세이지
- 4회 이상: 깊은 세이지 또는 올리브

### 표시 방식

- 숫자를 크게 표시
- 짧은 단계명 사용: 시작 전, 전환, 지속, 완성, 회복, 관계, 결정
- 접근성 label에는 전체 조합 표시: "건강, 시작 전 마찰, 2회"
- hover보다 모바일 tap/focus 상태를 우선 고려

### 모바일

- Top 3는 가로 스크롤 없이 한 화면에 들어오게 한다.
- 히트맵은 가로 스크롤 유지
- 안내 문구: "가로로 밀어 더 볼 수 있습니다."

## 14. 상태 배지 스타일

내부 상태는 유지하되 화면에는 더 부드러운 표현을 사용한다.

| 내부 상태 | 화면 표시 | 톤 |
| --- | --- | --- |
| 진행 전 | 아직 안 함 | 조용한 회색 |
| 진행 중 | 해보는 중 | 세이지 |
| 관찰 완료 | 살펴봄 | 딥 올리브 |
| 보류 | 잠시 멈춤 | 샌드/브라운 |

배지 원칙:

- 성공/실패처럼 읽히지 않게 한다.
- 상태는 평가가 아니라 현재 위치를 표시한다.
- 색상만으로 구분하지 않고 텍스트를 함께 표시한다.

## 15. 마이크로카피 톤 가이드

문구는 사용자를 몰아붙이지 않는다. 모든 문장은 "해야 한다"보다 "해볼 수 있다"에 가깝게 쓴다.

### 권장 문구

- 한 줄만 남겨도 충분합니다.
- 정확히 고르지 않아도 괜찮아요.
- 지금 가장 가까운 느낌을 골라보세요.
- 비슷한 순간이 쌓이면 자주 막히는 위치가 보입니다.
- 바로 해결하지 않아도 괜찮아요.
- 다음에 덜 버겁게 해볼 작은 방법을 정해보세요.
- 잘 안 되면 방법을 더 작게 줄여볼 수 있습니다.
- 기록은 이 기기 안에 저장됩니다.

### 피해야 할 문구

- 실패했습니다.
- 의지가 부족합니다.
- 문제 영역입니다.
- 반드시 고쳐야 합니다.
- 습관이 나쁩니다.
- 심리 진단 결과입니다.
- 불안을 치료합니다.
- 정신건강을 개선합니다.

## 16. 미세한 모션 가이드

모션은 감성적 완성도를 높이지만, 과하면 기록 앱의 조용함을 해친다.

사용 가능한 모션:

- 탭 전환: 120~180ms fade + slight translate
- 버튼 press: scale 0.98 정도
- 칩 선택: 배경색과 border 120ms 전환
- 저장 피드백: 아래에서 살짝 올라오며 나타나고 2~3초 후 사라짐
- 지도 타일: 처음 로드 시 아주 짧은 opacity transition

피해야 할 모션:

- 튀는 bounce
- 과한 confetti
- streak/게임화처럼 보이는 애니메이션
- 사용자를 평가하는 완료 연출

## 17. 데스크톱 미리보기 방식

마찰지도는 모바일 앱 중심으로 설계한다. 데스크톱에서는 대시보드를 펼치기보다 "모바일 앱 프레임 미리보기"처럼 보이게 하는 것이 좋다.

데스크톱 방향:

- 중앙에 최대 폭 430~480px의 앱 프레임
- 바깥 배경은 넓은 그라데이션 또는 조용한 톤
- 오른쪽/왼쪽에 마케팅 설명 패널을 추가하지 않는다.
- 개발/스토어 스크린샷을 찍기 쉬운 구조를 유지한다.

장점:

- Android 앱과 웹 미리보기 경험이 일관된다.
- 모바일 QA가 쉬워진다.
- Play Store 스크린샷 구성과도 잘 맞는다.

## 19. QA 체크리스트

### 기능 유지

- 막힌 순간 기록 추가
- 빈 입력/공백 입력 방지
- 빠른 예시 칩 입력 보조
- 기록 저장 후 피드백 표시
- 새로고침 또는 앱 재실행 후 기록 유지
- 기록 삭제 후 지도/요약 반영
- 자주 막힌 위치 Top 3 정확성
- 마찰 지도 개수 정확성
- 시도 카드 생성
- 시도 상태 변경
- 돌아보기 메모 저장
- 전체 기록 삭제
- 기록 내보내기
- 다크모드 유지

### 모바일 UI

- 한 화면에서 주요 목적이 바로 보이는가
- Bottom nav가 화면을 가리지 않는가
- 칩 버튼이 손가락으로 누르기 쉬운가
- 입력창이 너무 작거나 답답하지 않은가
- 지도 가로 스크롤이 자연스러운가
- 작은 기기에서도 버튼 텍스트가 잘리지 않는가
- Android WebView에서 폰트와 여백이 과하게 커지지 않는가

### 다크모드

- 배경과 카드 경계가 구분되는가
- 텍스트 대비가 충분한가
- 선택 칩과 상태 배지가 잘 보이는가
- 지도 강도 차이가 어둡게 뭉개지지 않는가

### 문구

- 치료, 진단, 의료처럼 보이는 문구가 없는가
- 실패, 의지 부족, 문제 영역 같은 표현이 불필요하게 노출되지 않는가
- 막힘, 자주 막힌 위치, 작게 시도하기, 돌아보기, 덜 버겁게 같은 친숙한 표현이 유지되는가
- 사용자가 기록을 통제할 수 있다는 신뢰 문구가 있는가

### 프리미엄 인상

- 과한 장식 없이 완성도 있게 보이는가
- 카드, 버튼, 칩, 배지가 같은 디자인 언어로 보이는가
- 여백이 충분한가
- 첫 화면이 앱의 목적을 3초 안에 전달하는가
- Play Store 스크린샷으로 써도 어색하지 않은가
