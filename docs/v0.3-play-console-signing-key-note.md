# v0.3 Play Console 서명 키 메모

## 현재 상황

Play Console에 AAB를 업로드할 때 아래 오류가 발생했다.

```text
Android App Bundle이 잘못된 키로 서명되었습니다.
제대로 된 서명 키로 App Bundle에 서명한 다음 다시 시도해 보세요.
```

Play Console이 기대하는 인증서 SHA1:

```text
F4:F2:19:CF:49:28:EC:CD:E3:D9:C2:13:77:0E:B6:DE:2A:21:5C:9E
```

현재 로컬 `android/release-upload-key.jks`로 서명한 AAB의 SHA1:

```text
02:6A:63:E2:2B:10:6B:44:06:FC:B6:07:DE:E1:09:62:2F:68:A1:2F
```

즉, 현재 로컬에서 만든 AAB는 형식은 맞지만 Play Console이 기대하는 업로드 키와 다르다.

## 생성된 파일

새로 만든 내부 테스트용 AAB:

```text
android/release/friction-map-v0.3.2-signed.aab
```

현재 로컬 업로드 키 인증서:

```text
android/release/friction-map-upload-certificate-sha1-026a.pem
```

주의: `.pem` 파일은 AAB 업로드 칸에 올리는 파일이 아니다. Play Console에서 업로드 키 변경 또는 재설정이 필요할 때 사용하는 인증서 파일이다.

## 해결 선택지

### 선택지 A. 원래 F4:F2 키를 찾는다

가장 빠른 해결책이다. 아래 지문을 가진 keystore 또는 jks 파일을 찾아야 한다.

```text
F4:F2:19:CF:49:28:EC:CD:E3:D9:C2:13:77:0E:B6:DE:2A:21:5C:9E
```

이 키를 찾으면 `android/keystore.properties`가 그 키를 바라보게 바꾼 뒤 AAB를 다시 빌드하면 된다.

### 선택지 B. Play Console에서 업로드 키 변경을 진행한다

F4:F2 키를 찾을 수 없다면 현재 로컬 키를 새 업로드 키로 등록해야 한다.

사용할 인증서:

```text
android/release/friction-map-upload-certificate-sha1-026a.pem
```

이 파일을 AAB 업로드 칸에 올리면 안 된다. Play Console의 서명 키 변경 또는 업로드 키 재설정 흐름에서 사용해야 한다.

## 다음에 AAB를 다시 만들 때

버전:

```text
versionCode 3
versionName 0.3.2
```

명령:

```powershell
pnpm.cmd build
npx cap sync android
cd android
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
.\gradlew.bat bundleRelease
```

출력 파일:

```text
android/app/build/outputs/bundle/release/app-release.aab
```

업로드용 복사본:

```text
android/release/friction-map-v0.3.2-signed.aab
```

## 업로드 전 확인

- Play Console이 기대하는 SHA1과 AAB 서명 SHA1이 일치해야 한다.
- `.pem` 파일은 AAB가 아니다.
- AAB 업로드 칸에는 `.aab` 파일만 올린다.
- 업로드 키 변경을 했다면 변경이 반영된 뒤 다시 AAB를 올린다.
