# 마찰지도 v0.2.1 UI/UX 개선 계획

## 1. 현재 UI/UX 문제 진단

현재 앱은 v0.2 기능 흐름이 모두 연결되어 있지만, 화면의 정보 구조가 기능 추가 순서에 가깝게 쌓여 있습니다. 사용자가 처음 들어왔을 때 “지금 무엇을 하면 되는지”는 보이지만, “기록하기 -> 패턴 보기 -> 실험하기”라는 전체 여정은 아직 충분히 선명하지 않습니다.

주요 문제는 다음과 같습니다.

- 첫 화면의 헤더가 앱의 철학은 잘 전달하지만, 다음 행동으로 이어지는 시각적 안내가 약합니다.
- 마찰 기록 입력 폼이 일반적인 form 구조라서 자기이해 도구보다는 관리 도구처럼 느껴질 수 있습니다.
- 감정, 생활 영역, 마찰 단계 선택이 select 중심이라 선택지가 가진 의미가 한눈에 드러나지 않습니다.
- 마찰 지도는 기능적으로 정확하지만 표처럼 보여서 “지도”나 “히트맵”의 감각이 약합니다.
- 반복 요약, 주간 리포트, 작은 실험 카드가 모두 카드형으로 쌓이며 정보 우선순위가 비슷해 보입니다.
- 작은 실험 작성 영역은 입력 항목이 많아 긴 양식처럼 느껴질 수 있습니다.
- 기록 목록은 중요하지만, 현재는 핵심 화면과 같은 비중으로 보여 앱 하단의 무게가 커질 수 있습니다.
- 모바일에서는 모든 섹션이 세로로 쌓이므로 흐름 안내와 섹션 간 구분이 더 중요합니다.

## 2. v0.2.1 디자인 목표

v0.2.1은 새로운 핵심 기능을 추가하지 않고, 기존 기능의 사용 흐름과 시각적 완성도를 높이는 버전입니다.

목표:

- 사용 흐름을 “기록하기 -> 패턴 보기 -> 실험하기”로 명확히 보여줍니다.
- 첫 화면에서 사용자가 바로 한 줄 기록을 시작할 수 있게 합니다.
- 선택 UI를 더 직관적인 칩, 세그먼트, 배지 중심으로 바꿉니다.
- 마찰 지도를 더 시각적인 히트맵처럼 보이게 합니다.
- 작은 실험 카드를 긴 양식이 아니라 차분한 실험 설계 도구처럼 보이게 합니다.
- 기록 목록은 보조 영역으로 낮춰 핵심 화면을 밀어내지 않게 합니다.
- 다크모드에서도 카드, 배지, 지도 셀의 대비를 안정적으로 유지합니다.
- 모바일에서 손가락으로 선택하기 쉽고, 섹션 간 흐름이 자연스럽게 이어지게 합니다.

## 3. 새 화면 정보 구조

권장 정보 구조:

```text
상단 헤더
-> 흐름 안내: 기록하기 / 패턴 보기 / 실험하기
-> 빠른 기록 카드
-> 패턴 보기 영역: 마찰 지도 + 반복 요약
-> 실험하기 영역: 실험 작성 + 실험 카드 목록
-> 보조 영역: 주간 리포트 + 기록 목록
```

v0.1의 핵심인 마찰 기록과 지도는 계속 위쪽에 둡니다. v0.2의 작은 실험 카드는 패턴을 확인한 뒤 자연스럽게 이어지는 다음 단계로 배치합니다.

## 4. 추천 화면 배치

데스크톱:

- 헤더는 넓고 얕게 유지합니다.
- 헤더 아래에 3단계 흐름 바를 둡니다.
- 첫 번째 주요 그리드:
  - 왼쪽: 빠른 기록 카드
  - 오른쪽: 마찰 지도
- 두 번째 주요 그리드:
  - 왼쪽: 반복 요약 + 주간 리포트
  - 오른쪽: 작은 실험 카드 작성
- 실험 목록은 폭이 넓은 단일 섹션 또는 2열 카드 그리드로 배치합니다.
- 기록 목록은 접기/펼치기 또는 보조 카드 느낌으로 아래에 둡니다.

모바일:

- 헤더 -> 흐름 안내 -> 빠른 기록 -> 마찰 지도 -> 요약 -> 실험 작성 -> 실험 목록 -> 기록 목록 순서로 한 열 배치합니다.
- 마찰 지도는 가로 스크롤을 유지하되, 셀의 최소 크기와 행/열 라벨의 가독성을 높입니다.
- 기록 목록은 화면 하단에 두고 카드 간 간격을 충분히 둡니다.

## 5. 컴포넌트 개선 목록

- `page.tsx`
  - 섹션 순서와 그리드 구조를 재정리합니다.
  - 흐름 안내 컴포넌트를 추가할지 검토합니다.
- `FrictionForm`
  - select UI를 칩/세그먼트 선택 UI로 바꿉니다.
  - 강도 선택을 현재 라디오 카드 방식에서 더 또렷한 세그먼트로 다듬습니다.
- `FrictionMap`
  - 표 기능은 유지하되 히트맵처럼 보이는 셀 스타일을 강화합니다.
  - 범례를 더 작고 명확하게 정리합니다.
- `FrictionSummary`
  - 세 카드의 정보 위계를 정리하고, Top 3 항목을 더 읽기 쉽게 만듭니다.
- `WeeklyReport`
  - 분석 대시보드보다 부드러운 리포트 카드처럼 다듬습니다.
- `ExperimentBuilder`
  - 긴 양식 느낌을 줄이고 입력을 작은 그룹으로 나눕니다.
  - 연결 기록 선택 영역을 더 눈에 띄게 만듭니다.
- `ExperimentList`
  - 실험 카드의 제목, 상태, 연결 기록, 가설/실험 내용을 더 명확하게 분리합니다.
  - 상태 컨트롤을 카드 상단의 조용한 세그먼트 또는 select로 유지하되 시각적 무게를 조절합니다.
- `FrictionList`
  - 보조 영역으로 시각적 무게를 낮춥니다.
  - 최신 기록 일부를 먼저 보여주고 전체 목록은 추후 접기 후보로 둡니다.
- `ThemeToggle`
  - 현재 동작은 유지하고, 헤더 안에서 더 자연스러운 위치와 크기로 조정합니다.

## 6. 입력 폼 개선 방향

마찰 기록 폼은 “작성해야 하는 양식”이 아니라 “막힌 순간을 살짝 내려놓는 공간”처럼 보여야 합니다.

개선 방향:

- textarea를 화면에서 가장 먼저 눈에 들어오는 입력 카드로 둡니다.
- 감정, 생활 영역, 마찰 단계는 select 대신 선택 칩으로 바꿉니다.
- 선택지는 한눈에 훑을 수 있어야 하며 모바일에서도 두세 줄로 자연스럽게 감싸지게 합니다.
- 강도는 1~5 세그먼트 선택으로 유지하되, 현재 선택값이 더 분명하게 보이게 합니다.
- 저장 버튼은 단순히 “기록 저장”보다 “마찰 기록 남기기”처럼 행동을 부드럽게 설명하는 문구를 검토합니다.
- 빈 입력 안내는 오류처럼 보이지 않게, 작은 도움말이나 `aria-live` 안내로 유지합니다.

주의:

- 입력 폼에 과한 설명을 넣지 않습니다.
- 기록을 잘해야 한다는 압박을 주지 않습니다.
- 선택 칩이 너무 많아 보이면 영역별로 적절한 줄바꿈과 간격을 둡니다.

## 7. 마찰 지도 개선 방향

마찰 지도는 v0.2.1의 핵심 시각 요소로 다듬습니다.

개선 방향:

- 표 구조와 접근성은 유지하되, 셀 배경 강도를 더 히트맵답게 조정합니다.
- 0개 셀은 거의 배경에 가까운 색으로 낮춥니다.
- 1개, 2~3개, 4개 이상 셀은 차분한 한 계열의 강도 차이를 사용하되 너무 자극적이지 않게 합니다.
- 행 라벨과 열 라벨의 sticky 처리는 모바일에서 필요 여부를 검토합니다.
- 지도 상단에는 “마찰이 자주 나타난 위치”라는 표현을 유지합니다.
- 범례는 작은 배지나 선형 스케일로 단순화합니다.
- 셀 안 숫자가 작아도 읽히도록 최소 크기와 폰트 대비를 보장합니다.

피할 것:

- “문제 영역” 같은 표현
- 지나치게 강한 빨간색 경고 팔레트
- 모바일에서 표가 페이지 전체 레이아웃을 깨뜨리는 구조

## 8. 작은 실험 카드 개선 방향

작은 실험 카드는 v0.2의 핵심 확장 기능입니다. 작성 폼은 길지만 사용자가 부담스럽게 느끼지 않도록 단계와 그룹을 나눕니다.

개선 방향:

- 실험 작성 영역을 “마찰 선택”, “가설”, “실험 설계”, “관찰 기준” 4개 그룹으로 나눕니다.
- 연결된 마찰 기록 선택을 카드형 preview처럼 보여줍니다.
- 실험 기간은 3일, 5일, 7일 세그먼트로 유지합니다.
- 성공 기준은 결과 판정이 아니라 관찰 기준이라는 설명을 작은 도움말로 둡니다.
- 예상대로 안 됐을 때의 해석은 “실험 단위를 조정하는 힌트”라는 방향으로 문구를 다듬습니다.
- 실험 카드 목록은 제목과 상태가 먼저 보이고, 세부 내용은 읽기 좋은 정보 블록으로 배치합니다.
- 상태 `관찰 완료`는 성공/실패 판정이 아니라 돌아볼 준비가 된 상태로 표현합니다.

## 9. 기록 목록 개선 방향

기록 목록은 중요한 데이터이지만 v0.2.1 화면에서는 보조 영역으로 둡니다.

개선 방향:

- 기록 목록 섹션의 시각적 무게를 낮춥니다.
- 최신순 정렬은 유지합니다.
- 카드에 배지는 유지하되 색을 과하게 늘리지 않습니다.
- 삭제 버튼은 조용한 텍스트 버튼으로 유지합니다.
- 추후 후보로 “최근 N개만 먼저 표시” 또는 “전체 기록 보기” 패턴을 고려합니다.

이번 단계에서는 새 기능을 만들지 않으므로 접기/검색/필터는 구현하지 않습니다.

## 10. 다크모드 개선 방향

다크모드는 단순 반전이 아니라 밤에 오래 봐도 피로하지 않은 자기이해 도구처럼 보여야 합니다.

개선 방향:

- 배경은 완전한 검정보다 깊은 slate 계열을 유지합니다.
- 카드 배경과 페이지 배경의 차이를 충분히 둡니다.
- 테두리는 너무 밝지 않게 조정합니다.
- 히트맵 셀은 다크모드에서 숫자 대비가 충분해야 합니다.
- 선택 칩, 세그먼트, 상태 배지는 다크모드에서도 선택 상태가 명확해야 합니다.
- focus ring은 유지합니다.

## 11. 모바일 UX 개선 방향

모바일에서는 정보량보다 흐름이 중요합니다.

개선 방향:

- 모든 주요 섹션은 한 열로 자연스럽게 쌓이게 합니다.
- 입력 칩은 최소 높이 40px 이상으로 눌리기 쉽게 합니다.
- 마찰 지도는 `overflow-x-auto`를 유지하고, 스크롤 영역이 분명하게 보이게 합니다.
- 작은 실험 폼은 긴 textarea가 너무 많이 이어지지 않도록 그룹 간 간격을 둡니다.
- 실험 카드의 상태 컨트롤과 삭제 버튼이 겹치지 않게 합니다.
- 기록 카드의 배지는 줄바꿈되어도 읽기 좋아야 합니다.
- iPhone SE, Pixel 7, iPad 크기에서 확인합니다.

## 12. 문구/마이크로카피 원칙

문구 원칙:

- 사용자를 평가하지 않습니다.
- 사용자를 진단하지 않습니다.
- 마찰을 의지나 성격의 문제로 해석하지 않습니다.
- “가설”, “관찰”, “패턴”, “작은 실험”이라는 언어를 유지합니다.
- 행동을 강요하지 않고 가볍게 권합니다.

권장 표현:

- “가볍게 관찰해볼 수 있습니다.”
- “작게 확인해볼 실험으로 바꿔보세요.”
- “반복되는 위치를 살펴봅니다.”
- “아직 기록이 없어도 괜찮습니다.”

피할 표현:

- “문제 영역”
- “실패 원인”
- “의지 부족”
- “나쁜 습관”
- “성공률”
- “점수”
- “달성률”

## 13. 구현 순서

1. `page.tsx` 섹션 구조를 “기록하기 -> 패턴 보기 -> 실험하기” 흐름으로 재배치합니다.
2. 헤더와 흐름 안내 영역을 정리합니다.
3. `FrictionForm`을 칩/세그먼트 중심 UI로 개선합니다.
4. `FrictionMap`을 히트맵 시각에 가깝게 다듬습니다.
5. `FrictionSummary`와 `WeeklyReport`의 시각적 우선순위를 정리합니다.
6. `ExperimentBuilder`를 그룹형 카드 폼으로 정리합니다.
7. `ExperimentList` 카드 레이아웃을 더 읽기 쉽게 다듬습니다.
8. `FrictionList`를 보조 영역으로 낮춥니다.
9. 다크모드 색상과 대비를 점검합니다.
10. 모바일 viewport QA를 진행합니다.
11. `pnpm build`를 실행합니다.

## 14. QA 체크리스트

- `pnpm build`가 성공합니다.
- 마찰 기록 추가/삭제가 그대로 동작합니다.
- 새로고침 후 마찰 기록과 실험 카드가 유지됩니다.
- 마찰 지도 개수가 기존과 동일하게 계산됩니다.
- 반복 요약이 기존과 동일하게 표시됩니다.
- 작은 실험 카드 생성, 상태 변경, 삭제가 그대로 동작합니다.
- 다크모드 토글과 새로고침 후 테마 유지가 동작합니다.
- 첫 화면에서 “기록하기 -> 패턴 보기 -> 실험하기” 흐름이 보입니다.
- 입력 칩과 세그먼트가 키보드와 포커스 스타일을 유지합니다.
- 마찰 지도는 모바일에서 가로 스크롤이 가능합니다.
- 작은 실험 카드가 긴 문장에서도 레이아웃을 깨지 않습니다.
- 기록 목록이 핵심 화면을 과하게 밀어내지 않습니다.
- iPhone SE, Pixel 7, iPad 크기에서 큰 겹침이 없습니다.
- 문구에 비난, 진단, 점수 경쟁 느낌이 없습니다.
