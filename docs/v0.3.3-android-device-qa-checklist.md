# 마찰지도 v0.3.3 Android 실기기 QA 체크리스트

## 1. 목적

자동 QA와 브라우저 테스트를 통과해도 Android 실기기에서는 다른 문제가 생길 수 있다. 이 문서는 Play Store 내부 테스트 또는 폐쇄 테스트 전에 실제 기기에서 확인해야 할 항목을 정리한다.

중점 확인 대상:

- 하단 탭과 키보드 충돌
- 작은 화면에서 칩과 버튼 터치성
- WebView 저장 안정성
- 다크모드와 시스템 글자 크기
- 기록 내보내기와 전체 삭제
- Android 뒤로가기 동작

## 2. 권장 기기 매트릭스

가능하면 아래 조합을 최소 3대 이상에서 확인한다.

| 구분 | 예시 |
| --- | --- |
| 작은 화면 | Galaxy S 계열, Pixel 작은 모델 |
| 큰 화면 | Galaxy Ultra 계열, Pixel Pro 계열 |
| 보급형 | 저사양 Android 기기 |
| 태블릿 | 가능하면 1대 |
| Android 버전 | Android 10 이상, 최신 버전 1대 포함 |

## 3. 설치와 실행

- AAB 또는 내부 테스트 링크로 설치한다.
- 앱이 첫 실행에서 흰 화면으로 멈추지 않는지 확인한다.
- 오프라인 상태에서도 첫 화면이 열리는지 확인한다.
- 앱을 완전히 종료한 뒤 다시 열어도 정상 진입하는지 확인한다.
- 앱 업데이트 후 기존 기록이 유지되는지 확인한다.

## 4. 첫 화면과 남기기 탭

- 앱을 열었을 때 “오늘 어디서 막혔나요?” 목적이 바로 보인다.
- 입력창이 화면 안에 자연스럽게 보인다.
- 예시 칩이 손가락으로 누르기 쉽다.
- 예시 칩을 누르면 텍스트와 기본 선택값이 함께 채워진다.
- 사용자가 예시 선택 후 감정/영역/단계/버거움을 다시 바꿀 수 있다.
- 키보드가 올라와도 저장 버튼을 찾을 수 있다.
- 하단 탭이 키보드와 심하게 겹치지 않는다.
- 저장 후 피드백 메시지가 보인다.

## 5. 지도 탭

- Top 3가 지도보다 먼저 보인다.
- 기록이 없을 때 빈 상태가 부드럽게 보인다.
- 기록이 1~2개일 때 과한 판단처럼 보이지 않는다.
- 지도는 가로로 밀어 볼 수 있다.
- 가로 스크롤이 페이지 전체를 깨뜨리지 않는다.
- 범례가 보인다.
- 숫자와 색 대비가 다크모드에서도 충분하다.

## 6. 시도 탭

- 기록이 없을 때 먼저 기록이 필요하다는 안내가 나온다.
- 기록이 있을 때 시도 카드 작성 흐름이 보인다.
- 입력 필드가 너무 긴 숙제처럼 느껴지지 않는다.
- 키보드가 올라와도 다음 입력칸과 저장 버튼을 찾을 수 있다.
- 상태 변경 버튼 또는 선택 UI가 터치하기 쉽다.
- 상태 변경 후 앱 재실행 시 상태가 유지된다.
- 연결된 기록을 삭제해도 시도 카드가 깨지지 않는다.

## 7. 돌아보기 탭

- 전체 기록 목록을 확인할 수 있다.
- 기록 삭제가 정상 동작한다.
- 삭제 후 앱 재실행 시 삭제된 기록이 다시 나타나지 않는다.
- 기록 내보내기 버튼이 보인다.
- 내보내기 파일이 생성된다.
- 파일 이름이 날짜를 포함한다.
- 모든 기록 삭제 전 확인 단계가 있다.
- 전체 삭제 후 기록과 시도 카드가 모두 비워진다.
- 개인정보 안내가 앱 안에서 확인된다.
- 앱 버전과 문의 이메일 자리가 보인다.

## 8. 저장 안정성

아래 순서로 확인한다.

```text
1. 기록 3개 생성
2. 시도 카드 1개 생성
3. 시도 상태 변경
4. 앱 종료
5. 기기 재부팅 또는 앱 강제 종료
6. 앱 재실행
7. 기록과 시도 카드가 유지되는지 확인
```

현재 저장 방식이 localStorage 기반이면 Android WebView 환경에서 장기 보존성을 추가로 관찰해야 한다. Capacitor Preferences 적용 전까지는 업데이트, 캐시 정리, 저장공간 부족 상황에서 기록 유지 여부를 주의해서 본다.

## 9. 다크모드와 접근성

- 앱 내부 다크모드 토글이 동작한다.
- 다크모드 설정이 앱 재실행 후 유지된다.
- Android 시스템 다크모드에서도 이상한 색 충돌이 없다.
- 시스템 글자 크기를 크게 했을 때 버튼 텍스트가 잘리지 않는다.
- 칩이 여러 줄로 자연스럽게 내려간다.
- 지도 셀이 너무 작아지지 않는다.
- 삭제 버튼이 과하게 위협적으로 보이지 않는다.

## 10. Android 뒤로가기

확인할 것:

- 뒤로가기를 눌렀을 때 앱이 갑자기 종료되어 혼란스럽지 않은지
- 하단 탭 이동 후 뒤로가기 동작이 예상 가능한지
- 입력 중 뒤로가기를 눌렀을 때 키보드가 먼저 내려가는지
- 저장되지 않은 입력이 갑자기 사라져 사용자가 당황하지 않는지

v0.3.3에서는 복잡한 뒤로가기 히스토리를 만들기보다, 예측 가능한 기본 동작을 우선한다.

## 11. Play Console 업로드 전 체크

- 올바른 서명 키로 AAB를 빌드했다.
- Play Console에 요구되는 업로드 인증서와 실제 AAB 서명이 일치한다.
- versionCode가 이전 빌드보다 증가했다.
- versionName이 의도한 값이다.
- 앱 이름과 패키지 이름이 일치한다.
- 내부 테스트 트랙에 올릴 AAB가 최신 빌드다.
- 경고가 가독화 파일 누락 정도인지, 업로드 차단 오류인지 구분한다.

## 12. 치명도 기준

### P0

- 앱이 실행되지 않는다.
- 기록이 저장되지 않는다.
- 앱 재실행 후 기록이 사라진다.
- 삭제한 기록이 다시 살아난다.
- Play Console에 AAB 업로드가 되지 않는다.

### P1

- 키보드가 저장 버튼을 가린다.
- 하단 탭이 눌리지 않는다.
- 지도 화면이 가로로 깨진다.
- 시도 카드 상태가 저장되지 않는다.
- 개인정보 안내를 찾기 어렵다.

### P2

- 일부 문구가 딱딱하다.
- 카드 간 여백이 좁다.
- 다크모드 대비가 살짝 낮다.
- 태블릿 화면에서 앱 프레임이 어색하다.

### P3

- 더 부드러운 애니메이션이 있으면 좋다.
- 아이콘이나 배지 스타일을 더 다듬을 수 있다.
- 스토어 스크린샷용 화면 구성이 더 필요하다.
