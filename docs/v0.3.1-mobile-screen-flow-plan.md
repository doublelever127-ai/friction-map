# 마찰지도 v0.3.1 모바일 화면 흐름 개선 계획

작성일: 2026-04-29  
브랜치: `v0.3.1-mobile-flow`  
목표: 긴 스크롤형 웹 대시보드를 실제 Android 앱처럼 한 화면 단위로 이해되는 하단 탭 구조로 바꾼다.

## 1. 왜 바꾸는가

v0.3.0 화면은 기능이 한 페이지에 길게 쌓여 있습니다. 웹에서는 자연스럽지만, 모바일에서는 처음 사용자가 아래로 계속 내려야 해서 “지금 무엇을 하면 되는지”가 흐려질 수 있습니다.

v0.3.1에서는 사용자가 한 화면에서 한 가지 일만 하도록 나눕니다.

```text
남기기 → 지도 → 시도 → 내 기록
```

## 2. 하단 탭 이름

최종 하단 탭은 다음 4개입니다.

```text
남기기 | 지도 | 시도 | 내 기록
```

| 탭 | 목적 | 이유 |
| --- | --- | --- |
| 남기기 | 오늘 막힌 순간을 한 줄 남김 | 첫 행동이 가장 분명함 |
| 지도 | 자주 막힌 위치를 봄 | 무엇을 보는지 바로 이해됨 |
| 시도 | 다음에 덜 버겁게 해볼 방법을 정함 | “바꾸기”보다 부담이 적음 |
| 내 기록 | 기록 확인, 정리, 개인정보 안내 | “관리”보다 개인 기록 앱에 가까움 |

## 3. 사용자용 용어

| 내부 개념 | 화면 표현 |
| --- | --- |
| 마찰 기록 | 막힌 순간 |
| emotion | 그때 느낌 |
| domain | 어느 쪽 일이었나요? |
| stage | 어디에서 막혔나요? |
| intensity | 얼마나 버거웠나요? |
| 작은 실험 카드 | 작게 바꿔보기 / 시도 카드 |
| 원인 가설 | 왜 막혔을까요? |
| 작은 실험 | 다음에는 어떻게 작게 해볼까요? |
| 성공 기준 | 어떻게 확인할까요? |
| 실패했을 때 해석 | 잘 안 됐을 땐 어떻게 볼까요? |
| 진행 전 | 아직 안 함 |
| 진행 중 | 해보는 중 |
| 관찰 완료 | 살펴봄 |
| 보류 | 잠시 멈춤 |

## 4. 화면 1: 남기기

목적: 오늘 막힌 순간을 한 줄 남깁니다.

상단 문구:

```text
오늘 어디서 막혔나요?

미뤘던 일, 부담스러운 답장, 시작하지 못한 순간을
한 줄만 남겨보세요.
```

포함할 요소:

- 한 줄 입력
- 빠른 예시 칩
- 그때 느낌
- 어느 쪽 일이었나요?
- 어디에서 막혔나요?
- 얼마나 버거웠나요?
- 기록하기 버튼
- 저장 직후 피드백
- 최근 기록 1~2개

빼야 할 요소:

- 전체 기록 목록
- 전체 마찰 지도
- 전체 반복 요약
- 시도 카드 작성 폼

저장 후 피드백:

```text
기록됐어요. 지도에 표시됐습니다.
```

보조 버튼:

```text
지도에서 보기
```

자동 이동은 하지 않습니다. 사용자가 직접 탭하거나 버튼을 눌러 이동하게 합니다.

## 5. 화면 2: 지도

목적: 내가 자주 막히는 위치를 봅니다.

상단 문구:

```text
자주 막힌 위치

어떤 일에서, 어떤 순간에 자주 막히는지
지도처럼 볼 수 있어요.
```

우선순위:

```text
1. 자주 막힌 위치 Top 3
2. 전체 마찰 지도
3. 반복 요약
4. 주간 리포트
```

기록 수별 문구:

| 기록 수 | 문구 |
| --- | --- |
| 0개 | 아직 지도가 비어 있어요. 막힌 순간을 하나 남기면 여기에 표시됩니다. |
| 1~2개 | 아직은 관찰을 시작한 단계예요. 비슷한 기록이 쌓이면 자주 막히는 위치가 더 선명해집니다. |
| 3개 이상 | 반복해서 보이는 위치를 살펴보세요. 바로 고치지 않아도 괜찮습니다. |

범례:

```text
기록 없음 / 가끔 보임 / 자주 보임 / 반복됨
```

모바일에서는 지도 위에 “가로로 밀어 더 볼 수 있습니다.” 안내를 둡니다.

## 6. 화면 3: 시도

목적: 자주 막힌 순간을 다음에 덜 버겁게 해볼 작은 방법으로 바꿉니다.

화면 제목은 “작게 바꿔보기”를 사용하고, 하단 탭은 “시도”를 사용합니다.

상단 문구:

```text
작게 바꿔보기

바로 해결하지 않아도 괜찮아요.
다음에 덜 버겁게 해볼 작은 방법을 정해보세요.
```

포함할 요소:

- 어떤 기록을 볼까요?
- 왜 막혔을까요?
- 다음에는 어떻게 작게 해볼까요?
- 어떻게 확인할까요?
- 시도 카드 만들기
- 내 시도 카드 목록

기록 수별 안내:

| 기록 수 | 안내 |
| --- | --- |
| 0개 | 아직 시도할 기록이 없어요. 먼저 막힌 순간을 하나 남겨보세요. |
| 1~2개 | 지금 바로 시도해봐도 되고, 비슷한 기록이 조금 더 쌓인 뒤 골라도 괜찮아요. |
| 3개 이상 | 지도에서 자주 보이는 위치를 골라 작게 시도해볼 수 있어요. |

## 7. 화면 4: 내 기록

목적: 내가 남긴 기록을 확인하고 정리합니다.

상단 문구:

```text
내 기록

내가 남긴 기록을 확인하고 정리할 수 있어요.
기록은 이 기기 안에 저장됩니다.
```

포함할 요소:

- 전체 기록 목록
- 기록 삭제
- 기록 내보내기
- 모든 기록 삭제
- 개인정보 안내
- 앱 버전
- 문의 이메일 자리
- 개인정보처리방침 링크

## 8. 구현 순서

1. 하단 탭 이름을 `남기기 | 지도 | 시도 | 내 기록`으로 확정한다.
2. `BottomNav` 컴포넌트의 문구를 수정한다.
3. `page.tsx`의 각 화면 상단 문구를 사용자 친화 표현으로 바꾼다.
4. 남기기 화면은 입력과 최근 기록 1~2개만 남긴다.
5. 지도 화면은 `Top 3 → 지도 → 요약` 순서로 정리한다.
6. 시도 화면은 “시도 카드” 언어를 적용한다.
7. 내 기록 화면은 전체 기록과 설정을 묶는다.
8. 저장 후 피드백에 “지도에서 보기” 버튼을 둔다.
9. 기존 localStorage와 계산 로직은 바꾸지 않는다.
10. `pnpm.cmd build`를 실행한다.

## 9. QA 체크리스트

### 기능

- [ ] 기록을 추가할 수 있다.
- [ ] 빠른 예시 칩이 텍스트와 기본 선택값을 함께 채운다.
- [ ] 빈 입력과 공백 입력은 저장되지 않는다.
- [ ] 저장 후 피드백이 보인다.
- [ ] “지도에서 보기” 버튼으로 지도 화면에 갈 수 있다.
- [ ] 새로고침 후 기록이 유지된다.
- [ ] 지도 개수가 맞다.
- [ ] Top 3가 맞다.
- [ ] 시도 카드를 만들 수 있다.
- [ ] 시도 카드 상태를 바꿀 수 있다.
- [ ] 시도 카드를 삭제할 수 있다.
- [ ] 내 기록에서 전체 기록을 볼 수 있다.
- [ ] 기록 내보내기가 가능하다.
- [ ] 모든 기록 삭제가 상태와 저장소에 반영된다.

### 모바일

- [ ] 첫 화면에서 입력 경험이 바로 보인다.
- [ ] 하단 탭이 화면 밖으로 밀리지 않는다.
- [ ] 칩 버튼이 손가락으로 누르기 쉽다.
- [ ] 지도는 가로 스크롤이 가능하다.
- [ ] 시도 카드가 화면 밖으로 튀지 않는다.
- [ ] 내 기록 화면의 삭제/내보내기 버튼이 명확하다.

### 문구

- [ ] “실패”, “의지 부족”, “문제 영역” 같은 표현이 불필요하게 보이지 않는다.
- [ ] 첫 화면은 “막힘” 중심으로 이해된다.
- [ ] “가설/실험/관찰”이 너무 딱딱하게 노출되지 않는다.
- [ ] 치료나 진단 앱처럼 보이는 문구가 없다.
