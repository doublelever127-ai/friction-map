# 마찰지도 v0.2.2 사용자 테스트 기반 개선 계획

## 문서 목적

이 문서는 v0.2.1 이후 사용자 테스트에서 나온 관찰을 바탕으로, v0.2.2에서 이해도와 사용성을 높이기 위한 개선 범위를 정리합니다.

이번 요청에는 실제 사용자 테스트 원문이 아직 붙어 있지 않았습니다. 따라서 아래 내용은 현재 v0.2.1 흐름, 사용자 여정 점검 문서, 모바일 QA에서 드러난 가능성 높은 혼란을 기준으로 작성한 개선 계획 초안입니다. 실제 테스트 메모가 추가되면 항목별 우선순위와 표현을 다시 조정합니다.

v0.2.2의 기준은 새 기능 확장보다 “처음 사용자가 더 빨리 이해하고, 더 편하게 첫 기록을 남기고, 기록이 지도와 실험으로 이어지는 흐름을 자연스럽게 보는 것”입니다.

## 1. 가장 자주 나온 혼란

예상되는 주요 혼란은 다음입니다.

- “마찰”이 할 일, 감정, 습관 중 무엇을 뜻하는지 처음에는 바로 잡히지 않을 수 있습니다.
- 예시 미리보기가 실제 저장된 기록인지 샘플인지 헷갈릴 수 있습니다.
- 첫 기록을 저장한 뒤 화면에서 어디가 달라졌는지 놓칠 수 있습니다.
- 마찰 지도에서 행과 열, 숫자, 색상 강도의 의미를 한 번에 이해하기 어려울 수 있습니다.
- 작은 실험 카드가 첫 기록 직후 바로 해야 하는 과제처럼 보일 수 있습니다.
- 모바일에서는 지도와 실험 카드가 길어져 핵심 흐름을 끝까지 따라가기 어려울 수 있습니다.

v0.2.2에서는 이 혼란을 줄이기 위해 각 섹션이 다음 행동을 짧게 안내하도록 조정합니다.

## 2. 첫 사용 경험 문제

현재 v0.2.1은 첫 화면에서 “오늘 어디서 막혔나요?”와 예시 칩을 제공해 첫 기록으로 진입하기 쉬워졌습니다. 다만 첫 사용자는 아직 다음 질문을 가질 수 있습니다.

- 이 기록이 어디에 쓰이는가?
- 선택 칩은 반드시 정확하게 골라야 하는가?
- 첫 기록 하나만으로도 의미가 있는가?
- 저장 후 무엇을 봐야 하는가?

개선 방향:

- 첫 기록 전에는 “정확한 분류보다 한 줄을 남기는 것이 먼저”라는 안내를 더 가볍게 보여줍니다.
- 첫 저장 직후 피드백을 입력 카드 근처와 패턴 영역 사이에서 더 잘 보이게 합니다.
- 기록 1개 상태에서는 “패턴을 정하는 단계가 아니라 관찰을 시작하는 단계”라는 안내를 유지합니다.
- 기록 3개 이상 상태에서는 패턴 보기 영역으로 자연스럽게 시선이 가도록 강조합니다.

## 3. 입력 폼 문제

입력 폼은 v0.2.1에서 선택 칩 중심으로 개선되었지만, 선택지가 많아 모바일에서 길어 보일 수 있습니다.

개선 방향:

- 빠른 예시 칩은 유지하되, 클릭하면 텍스트와 기본 선택값이 함께 채워진다는 점을 더 자연스럽게 느끼게 합니다.
- 감정, 생활 영역, 마찰 단계 선택은 “정답 고르기”가 아니라 “지금 가장 가까운 표현 고르기”로 안내합니다.
- 칩 버튼의 터치 영역과 줄바꿈은 계속 유지합니다.
- 빈 입력 안내는 오류처럼 보이지 않게 부드러운 안내로 유지합니다.
- 저장 후 메시지는 “마찰이 지도에 표시됐다”는 의미를 짧게 전달합니다.

v0.2.2에서 입력 폼의 목표는 사용자가 45초 안에 첫 기록을 남길 수 있게 하는 것입니다.

## 4. 마찰 지도 이해 문제

마찰 지도는 앱의 핵심 가치이지만, 처음 보는 사용자는 표처럼 받아들일 수 있습니다.

가능한 혼란:

- 행과 열이 무엇을 뜻하는지 바로 보이지 않을 수 있습니다.
- 숫자가 기록 수라는 점은 알 수 있지만, 왜 중요한지 약할 수 있습니다.
- 색이 진한 셀을 어떻게 해석해야 할지 모를 수 있습니다.
- 모바일에서 가로 스크롤이 가능하다는 사실을 놓칠 수 있습니다.

개선 방향:

- 지도 상단의 Top 3 영역을 더 짧고 명확하게 유지합니다.
- 범례는 계속 노출하되, “기록 없음”, “가끔 나타남”, “자주 나타남”, “반복 신호” 문구를 유지합니다.
- 모바일에서는 “가로로 밀어 더 보기” 같은 짧은 보조 문구를 검토합니다.
- 첫 기록 직후에는 방금 반영된 위치를 사용자가 알아보기 쉽게 하는 표현을 검토합니다.
- 지도는 평가가 아니라 관찰을 돕는 화면이라는 톤을 유지합니다.

## 5. 작은 실험 카드 문제

작은 실험 카드는 v0.2의 핵심이지만, 처음 사용하는 사람에게는 입력 항목이 많게 느껴질 수 있습니다.

가능한 혼란:

- 첫 기록 직후 바로 실험을 만들어야 하는 것처럼 느낄 수 있습니다.
- 원인 가설을 적는 일이 부담스러울 수 있습니다.
- 관찰 기준과 예상대로 안 됐을 때의 해석이 낯설 수 있습니다.
- 상태 변경이 완료 판정처럼 보일 수 있습니다.

개선 방향:

- 기록 0개 상태에서는 실험 작성 폼보다 “먼저 마찰 기록이 필요하다”는 빈 상태를 유지합니다.
- 기록 1~2개 상태에서는 “지금 바로 실험으로 바꿔도 되고, 조금 더 모아도 괜찮다”는 안내를 유지합니다.
- 기록 3개 이상 상태에서는 반복 요약이나 지도의 자주 나타난 위치에서 고르도록 안내합니다.
- “관찰 기준”과 “상태”는 평가가 아니라 지금 어디까지 살펴봤는지 표시하는 용도로 설명합니다.
- 실험 카드는 해결책보다 “가볍게 확인해볼 조건”이라는 표현을 유지합니다.

## 6. 모바일 사용성 문제

모바일에서는 모든 섹션이 한 열로 쌓이기 때문에 정보량이 많게 느껴질 수 있습니다.

가능한 문제:

- 첫 화면에서 입력 폼까지 내려가는 길이 길게 느껴질 수 있습니다.
- 칩 버튼이 여러 줄로 늘어나 선택 영역이 커질 수 있습니다.
- 마찰 지도는 가로 스크롤이 가능하지만, 사용자가 알아차리지 못할 수 있습니다.
- 작은 실험 카드의 textarea와 상태 칩이 이어져 화면이 길어질 수 있습니다.
- 긴 기록 문장이나 실험 문장이 카드 폭을 밀 수 있습니다.

개선 방향:

- 모바일에서 칩 버튼의 터치 크기는 유지하되, 줄 간격을 안정적으로 유지합니다.
- 카드 내부에는 `min-width` 방어와 줄바꿈을 계속 적용합니다.
- 마찰 지도에는 가로 스크롤 안내를 검토합니다.
- 작은 실험 카드는 각 단계의 제목과 설명을 짧게 유지합니다.
- 기록 목록은 핵심 흐름을 지나 하단에서 보조적으로 확인하게 유지합니다.

## 7. 우선순위 높은 수정

v0.2.2에서 우선순위가 높은 수정은 다음입니다.

1. 첫 저장 직후 피드백 가시성 강화
   - 사용자가 기록이 저장되고 지도에 반영되었다는 사실을 바로 이해하게 합니다.

2. 예시 미리보기와 실제 기록 구분 강화
   - “예시”, “저장되지 않음” 문구를 계속 명확하게 유지합니다.

3. 마찰 지도 읽는 법 보강
   - Top 3, 범례, 가로 스크롤 안내를 통해 지도를 처음 봐도 어디를 볼지 알게 합니다.

4. 작은 실험 카드 진입 부담 완화
   - 기록 개수에 따라 실험 섹션의 안내 강도를 조절합니다.

5. 모바일 카드 폭 안정성 유지
   - 긴 문장, 배지, 칩이 화면 밖으로 밀리지 않게 계속 점검합니다.

## 8. v0.2.2에 넣을 개선

v0.2.2는 이해도와 사용성 개선에 집중합니다.

- 첫 기록 전 안내 문구 다듬기
- 첫 기록 저장 후 피드백 위치와 시각적 무게 조정
- 예시 미리보기의 “저장되지 않음” 문구 강화
- 마찰 지도 Top 3와 범례의 설명 간결화
- 모바일 지도 가로 스크롤 안내 추가 검토
- 작은 실험 카드 섹션의 기록 개수별 안내 유지 및 문구 정리
- 관찰 기준, 가설, 실험 상태 문구를 더 부드럽게 정리
- 모바일에서 칩, 버튼, 카드, 긴 텍스트가 안정적으로 보이는지 QA

## 9. v0.2.2에서 미룰 개선

다음 항목은 v0.2.2 범위를 넘어설 수 있으므로 뒤로 미룹니다.

- 계정, 로그인, 서버 저장
- 데이터베이스 연동
- AI 조언 생성
- 기록 검색과 고급 필터
- 실험 템플릿 자동 추천
- 알림, 리마인더
- streak, 점수 경쟁, 달성률
- 통계 차트의 대규모 확장
- 공유 기능
- 여러 기기 동기화

v0.2.2에서는 앱의 방향을 넓히기보다 첫 사용 흐름을 더 선명하게 만드는 데 집중합니다.

## 10. 테스트 체크리스트

### 빌드와 기본 안정성

- `pnpm.cmd build`가 성공합니다.
- 서버 렌더링 중 `window` 또는 `localStorage` 오류가 없습니다.
- 브라우저 콘솔에 치명적인 오류가 없습니다.

### 첫 사용 흐름

- 기록이 0개일 때 예시 미리보기가 보입니다.
- 예시 미리보기에는 실제 저장되지 않는다는 안내가 보입니다.
- 사용자는 45초 안에 첫 마찰 기록을 남길 수 있습니다.
- 첫 기록 저장 후 피드백 메시지가 보입니다.
- 첫 기록 후 샘플 미리보기가 핵심 영역을 차지하지 않습니다.

### 입력 폼

- 빠른 예시 칩을 누르면 텍스트와 기본 선택값이 채워집니다.
- 사용자는 예시 선택 후 감정, 생활 영역, 마찰 단계, 강도를 다시 바꿀 수 있습니다.
- 빈 입력과 공백 입력은 저장되지 않습니다.
- 칩 버튼은 모바일에서 누르기 쉽습니다.

### 마찰 지도

- 기록 수가 지도 셀에 정확히 반영됩니다.
- Top 3가 생활 영역과 마찰 단계 조합 기준으로 표시됩니다.
- 범례가 보입니다.
- 모바일에서 지도는 가로 스크롤이 가능합니다.
- 지도 문구는 평가나 확정처럼 보이지 않습니다.

### 반복 요약

- 생활 영역, 마찰 단계, 감정 Top 3가 기록 개수와 맞습니다.
- 기록이 없을 때 빈 상태가 부드럽게 보입니다.
- 요약은 패턴을 보기 위한 힌트라는 표현을 유지합니다.

### 작은 실험 카드

- 기록이 0개일 때 실험 작성 폼이 강하게 앞서지 않습니다.
- 기록이 1~2개일 때 바로 실험으로 바꿔도 되고 더 모아도 된다는 안내가 보입니다.
- 기록이 3개 이상일 때 반복 요약이나 지도에서 골라볼 수 있다는 안내가 보입니다.
- 실험 카드 생성, 상태 변경, 삭제가 동작합니다.
- 새로고침 후 실험 카드와 상태가 유지됩니다.
- 연결된 마찰 기록이 삭제되어도 실험 카드가 깨지지 않습니다.

### 모바일 QA

- iPhone SE 크기에서 입력 폼이 화면 밖으로 튀지 않습니다.
- Pixel 7 크기에서 칩과 버튼이 누르기 쉽습니다.
- iPad 크기에서 2열 또는 3열 배치가 자연스럽습니다.
- 작은 실험 카드의 긴 문장이 카드 폭을 밀지 않습니다.
- 다크모드에서 텍스트 대비가 충분합니다.

### 문구 점검

- 사용자를 비난하는 문구가 없습니다.
- 사용자를 진단하는 문구가 없습니다.
- “마찰”, “패턴”, “가설”, “실험”, “관찰”이라는 언어가 유지됩니다.
- “기록하기 → 패턴 보기 → 실험하기” 흐름이 유지됩니다.
