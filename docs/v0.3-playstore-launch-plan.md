# 마찰지도 v0.3 Google Play 출시 계획

작성일: 2026-04-29  
목표 버전: v0.3 Android closed test 준비

## 1. 출시 목적

마찰지도 v0.3의 목적은 현재 Next.js 기반 웹앱을 Android 앱으로 패키징하고, Google Play Store 폐쇄 테스트에 올려 실제 사용자 피드백을 받는 것이다.

이번 단계의 목표는 정식 공개 출시가 아니라, 아래 흐름을 검증하는 것이다.

```text
앱 설치
→ 첫 화면 이해
→ 막힌 순간 한 줄 기록
→ 자주 막힌 위치 확인
→ 작게 바꿔보기 카드 생성
→ 앱 재실행 후 기록 유지
→ 사용자 피드백 수집
```

v0.3은 기능을 크게 늘리는 버전이 아니다. 출시 가능한 앱으로 신뢰감을 만들기 위해 저장 안정성, 데이터 통제, 개인정보 안내, Android 패키징을 정리하는 버전이다.

## 2. 앱 포지셔닝

마찰지도는 정신건강 치료 앱이나 심리 진단 앱이 아니다. 할 일 관리 앱이나 습관 점수 앱도 아니다.

추천 포지셔닝:

```text
자꾸 막히는 순간을 한 줄로 남기고,
반복되는 패턴을 보는 local-first 개인 기록 앱
```

스토어에서 강조할 가치:

- 할 일을 더 늘리지 않는다.
- 사용자가 자주 막히는 순간을 조용히 보여준다.
- 기록은 서버가 아니라 기기 안에 저장된다.
- 기록이 쌓이면 어느 쪽 일에서, 어느 순간에 자주 막히는지 볼 수 있다.
- 반복되는 막힘 하나를 다음에 덜 버겁게 해볼 작은 방법으로 바꿀 수 있다.

추천 카테고리 후보:

- 생산성
- 라이프스타일

앱 이름은 `마찰지도`를 유지한다. 다만 첫 화면과 스토어 설명에서는 `마찰`보다 `막힘`을 먼저 설명해 처음 보는 사람이 바로 이해할 수 있게 한다.

## 5. 출시 전 추가해야 할 기능

### 데이터 전체 삭제

사용자가 앱 안에서 자기 기록을 모두 지울 수 있어야 한다.

필요 동작:

- 모든 마찰 기록 삭제
- 모든 작게 바꿔보기 카드 삭제
- 삭제 후 화면 상태 즉시 초기화
- 앱 재실행 후에도 삭제 상태 유지
- 삭제 전 확인 단계 제공

추천 문구:

```text
마찰지도는 기록을 이 기기 안에 저장합니다.
필요할 때 모든 기록을 지울 수 있습니다.
```

버튼 문구:

```text
모든 기록 삭제
```

확인 문구:

```text
삭제하면 이 기기 안의 기록이 사라집니다. 계속할까요?
```

### 데이터 내보내기

사용자가 자신의 기록을 소유한다는 느낌이 중요하다. 첫 버전은 JSON 내보내기로 충분하다.

필요 동작:

- 마찰 기록과 작게 바꿔보기 카드를 하나의 JSON 파일로 내보내기
- 파일명 예: `friction-map-export-YYYY-MM-DD.json`
- 서버 전송 없이 기기 안에서 파일 생성
- 데이터가 없을 때 부드러운 안내 표시

추천 문구:

```text
내 기록을 파일로 저장해둘 수 있습니다.
```

### 개인정보 안내

앱 안에서 기록이 어떻게 저장되는지 짧고 명확하게 알려야 한다.

포함할 내용:

- 기록은 이 기기 안에 저장된다.
- 서버로 전송하지 않는다.
- 로그인 계정을 만들지 않는다.
- 광고 SDK를 사용하지 않는다.
- 분석 SDK를 사용하지 않는다.
- 사용자는 기록을 삭제하거나 내보낼 수 있다.
- 개인정보처리방침 URL을 확인할 수 있다.

### 앱 버전 표시

테스터 피드백과 문의 대응을 위해 앱 안에 버전을 표시한다.

추천 위치:

- 하단 정보 영역
- 설정/기록 관리 섹션

표시 예:

```text
마찰지도 v0.3.0
```

### 지원 이메일 표시

Google Play 등록과 사용자 문의 대응을 위해 지원 이메일을 정해야 한다.

표시 예:

```text
문의: your-email@example.com
```

실제 출시 전에는 사용 가능한 이메일로 교체한다.

## 6. Android 패키징 방식

후보:

| 방식 | 장점 | 단점 | 판단 |
| --- | --- | --- | --- |
| Capacitor | 현재 Next.js UI 재사용 가능, Android Studio 프로젝트 생성 가능 | 저장소/정적 export 점검 필요 | 추천 |
| React Native/Expo 재작성 | 네이티브 경험을 세밀하게 만들 수 있음 | 재작성 비용이 큼 | 보류 |
| TWA/PWA | 웹 배포 기반으로 빠르게 가능 | 로컬 앱 같은 신뢰감과 저장 안정성 점검 필요 | 보류 |
| Kotlin 네이티브 | Android 최적화 가능 | 현재 앱을 거의 다시 만들어야 함 | 비추천 |

v0.3에서는 Capacitor를 기준으로 진행한다.

## 7. Capacitor 추천 이유

마찰지도는 현재 다음 특성을 갖고 있다.

- 서버 기능 없음
- 로그인 없음
- AI API 없음
- 대부분 클라이언트 UI와 로컬 저장 중심
- 모바일에서 짧게 자주 쓰는 기록 앱

따라서 기존 웹앱을 재작성하기보다 Capacitor로 Android 앱 컨테이너를 만드는 것이 현실적이다.

Capacitor 적용 흐름:

```text
Next.js 정적 export 점검
→ Capacitor 설치
→ Android 플랫폼 추가
→ 웹 빌드
→ Capacitor sync
→ Android Studio에서 실행
→ AAB 빌드
→ Play Console 업로드
```

예상 명령:

```powershell
pnpm add @capacitor/core @capacitor/cli
pnpm add @capacitor/android
npx cap init
npx cap add android
pnpm.cmd build
npx cap sync android
npx cap open android
```

추천 앱 ID:

```text
com.dabin.frictionmap
```

## 8. 저장소 개선 계획

현재 v0.2.2는 `localStorage`를 사용한다.

웹에서는 괜찮지만 Android 앱에서는 저장 안정성을 더 신중하게 봐야 한다. Capacitor Preferences는 가벼운 키-값 저장을 위한 공식 플러그인으로, 모바일 앱 패키징 단계에서 적용하기 좋다.

v0.3 저장소 개선 순서:

1. `src/lib/storageAdapter.ts` 생성
2. 웹 환경에서는 기존 `localStorage` 사용
3. `frictionStorage.ts`와 `experimentStorage.ts`가 직접 `localStorage`를 만지지 않게 변경
4. 기존 저장 키 유지
5. Android 전환 단계에서 Capacitor Preferences 연결

유지할 저장 키:

```text
friction-map.logs.v1
friction-map.experiments.v1
```

저장소 어댑터 목표:

```text
웹: localStorage
Android 앱: Capacitor Preferences
서버 렌더링 중: 안전하게 빈 값 처리
```

기존 데이터는 깨지지 않아야 한다.

## 9. 개인정보/데이터 안전성 방향

마찰지도는 사용자가 사적인 내용을 적을 수 있는 앱이다. 서버로 보내지 않더라도 개인정보 안내와 Data Safety 작성이 중요하다.

첫 출시 버전의 데이터 방향:

```text
서버 없음
로그인 없음
광고 없음
분석 SDK 없음
사용자 기록은 기기 안에 저장
사용자가 전체 삭제 가능
사용자가 내보내기 가능
```

Data Safety 작성 방향:

- 앱이 사용자 기록을 외부 서버로 전송하지 않는다고 설명한다.
- 광고 목적 수집이 없다고 설명한다.
- 계정 생성이 없다고 설명한다.
- 앱 내 기록 삭제 기능을 제공한다고 설명한다.
- SDK를 추가하지 않는 한 데이터 공유가 없다고 설명한다.

주의:

나중에 Firebase Analytics, Sentry, 광고 SDK, 로그인, AI API, 클라우드 동기화를 추가하면 개인정보처리방침과 Data Safety를 다시 작성해야 한다.

## 11. 폐쇄 테스트 계획

폐쇄 테스트의 목적은 Play Console 절차를 통과하는 것뿐 아니라, 실제 사용자가 첫 기록과 작게 바꿔보기 흐름을 이해하는지 확인하는 것이다.

테스트 목표:

- 앱 설치가 문제 없는지 확인
- 첫 화면 목적이 이해되는지 확인
- 45초 안에 첫 기록이 가능한지 확인
- 기록 후 지도에 반영되는지 이해하는지 확인
- 작게 바꿔보기 카드가 부담스럽지 않은지 확인
- 앱 재실행 후 기록이 유지되는지 확인
- 데이터 삭제/내보내기 문구가 신뢰감을 주는지 확인

14일 테스트 운영안:

| 기간 | 미션 |
| --- | --- |
| 1일차 | 앱 설치 후 첫 기록 남기기 |
| 2~3일차 | 막힌 순간을 하루 1개 정도 기록 |
| 4일차 | 자주 막힌 위치 지도 확인 |
| 5~7일차 | 작게 바꿔보기 카드 1개 만들기 |
| 8~10일차 | 카드 상태를 바꿔보기 |
| 11~13일차 | 기록을 조금 더 쌓고 패턴 확인 |
| 14일차 | 짧은 리뷰 설문 작성 |

```text
- 자주 막힌 위치 지도
- 반복 패턴 요약
- 작게 바꿔보기 카드
- 로컬 저장
- 기록 내보내기
- 모든 기록 삭제

개인정보
마찰지도는 로그인 없이 사용할 수 있습니다.
사용자가 남긴 기록은 기기 안에 저장됩니다.
서버로 전송하지 않습니다.
```

## 14. 앱 스크린샷 구성안

스토어 스크린샷은 기능을 많이 보여주기보다 첫 가치 경험을 보여준다.

추천 순서:

1. 첫 화면
   - 문구: `오늘 어디서 막혔나요?`
   - 보여줄 것: 한 줄 기록 입력 카드

2. 빠른 기록
   - 문구: `한 줄만 남겨도 충분합니다`
   - 보여줄 것: 예시 칩, 느낌/영역/막힌 위치 선택

3. 자주 막힌 위치
   - 문구: `반복되는 막힘을 지도처럼 봅니다`
   - 보여줄 것: 히트맵과 Top 3

4. 작게 바꿔보기
   - 문구: `다음에 덜 버겁게 해볼 작은 방법`
   - 보여줄 것: 작게 바꿔보기 카드

5. 내 기록 관리
   - 문구: `기록은 이 기기 안에 저장됩니다`
   - 보여줄 것: 삭제/내보내기/개인정보 안내

다크모드 스크린샷도 1~2장 준비하면 앱의 분위기를 보여주기 좋다.
