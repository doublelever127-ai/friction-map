# 사용자용 용어 사전

마찰지도는 내부적으로 "마찰", "가설", "실험", "관찰"이라는 개념을 유지합니다. 다만 처음 사용하는 사람에게는 이 단어들이 조금 추상적으로 느껴질 수 있으므로, 첫 사용 화면과 입력 흐름에서는 더 익숙한 표현으로 풀어 씁니다.

이 문서는 앞으로 UI 문구를 작성할 때 사용할 사용자 친화 언어 기준입니다.

## 1. 현재 용어와 사용자 친화 용어 비교

| 현재 용어 | 사용자 친화 용어 | 사용 기준 |
| --- | --- | --- |
| 마찰 | 막힘, 막혔던 순간 | 앱 이름과 핵심 설명에서는 "마찰"을 유지하되, 첫 화면과 입력 질문에서는 "막힘"을 먼저 사용합니다. |
| 마찰 기록 | 막혔던 순간, 한 줄 기록 | 버튼이나 목록 제목처럼 기능명이 필요한 곳에서는 "마찰 기록"을 사용합니다. |
| 마찰 단계 | 어디에서 막혔나요? | 입력 폼에서는 질문형 문구로 풀어 씁니다. |
| 생활 영역 | 어떤 일에서 막혔나요? | "일/공부/건강" 같은 선택지 앞에서는 더 일상적인 질문을 사용합니다. |
| 감정 | 그때 어떤 느낌이었나요? | 감정을 진단하지 않고 상황의 단서로 다룹니다. |
| 강도 | 얼마나 버거웠나요? | 숫자 선택 앞에서는 "강도"보다 사용자가 느낀 부담을 묻는 표현을 사용합니다. |
| 가설 | 왜 그랬을까요? | 카드 제목이나 단계 이름에서는 "원인 가설"을 유지할 수 있지만, 입력 질문에서는 쉽게 풀어 씁니다. |
| 실험 | 작게 바꿔보기 | 기능명은 "작은 실험 카드"를 유지하되, 첫 안내에서는 "작게 바꿔보기"를 함께 사용합니다. |
| 관찰 | 가볍게 살펴보기, 확인해보기 | 상태나 안내 문구에서 평가가 아니라 돌아보는 느낌을 줍니다. |
| 패턴 | 자주 보이는 흐름, 반복되는 위치 | 요약과 지도에서는 "패턴"을 유지하되 보조 설명으로 풀어 씁니다. |

## 2. 앱 이름은 유지할지 검토

앱 이름 "마찰지도"는 유지합니다.

이유:

- 제품의 핵심 정체성인 "막히는 위치를 지도처럼 본다"는 의미가 선명합니다.
- 할 일 앱, 습관 앱, 감정 일기와 다른 위치를 만들어줍니다.
- "마찰"이라는 단어가 처음에는 낯설 수 있지만, 앱 안에서 "하려고 했지만 이상하게 막혔던 순간"으로 풀어 설명하면 충분히 이해할 수 있습니다.

보완 방향:

- 첫 화면에서는 앱 이름 바로 아래에 쉬운 설명을 붙입니다.
- "마찰"을 처음부터 반복하기보다 "막힘"으로 시작하고, 그다음 "이 앱에서는 이런 막힘을 마찰이라고 부릅니다"에 가깝게 안내합니다.
- 기능명에는 "마찰 기록", "마찰 지도"를 유지하되, 입력 질문은 생활 언어로 씁니다.

## 3. 첫 화면에서 쓸 핵심 문구

첫 화면은 사용자가 바로 "무엇을 적으면 되는지" 이해하게 만드는 곳입니다. 추상적인 제품 설명보다 짧은 질문과 쉬운 정의를 먼저 보여줍니다.

권장 문구:

- "오늘 어디서 막혔나요?"
- "마찰은 하려고 했지만 이상하게 막혔던 순간입니다."
- "거창한 기록이 아니어도 괜찮습니다. 한 줄만 남겨도 충분합니다."
- "할 일을 더 늘리지 않습니다. 당신을 막는 마찰을 발견합니다."
- "한 줄의 막힘이 쌓이면, 반복되는 위치가 보입니다."

피해야 할 방향:

- 첫 화면에서 "자기이해 도구", "행동 단계 분석", "원인 가설" 같은 추상어를 길게 설명하지 않습니다.
- 사용자가 뭔가 잘못했다는 느낌을 주는 표현을 쓰지 않습니다.

## 4. 입력 폼에서 쓸 질문형 문구

입력 폼은 "양식 작성"보다 "가볍게 남기기"처럼 느껴져야 합니다. 필드명만 나열하기보다 질문형 문구를 사용합니다.

권장 질문:

- "무슨 일이 막혔나요?"
- "한 줄로 적어볼까요?"
- "그때 어떤 느낌이었나요?"
- "어떤 일에서 생긴 막힘인가요?"
- "어디에서 막혔나요?"
- "얼마나 버거웠나요?"

현재 선택지와 연결하는 방식:

- 감정: "그때 어떤 느낌이었나요?"
- 생활 영역: "어떤 일에서 생긴 막힘인가요?"
- 마찰 단계: "어디에서 막혔나요?"
- 강도: "얼마나 버거웠나요?"

빠른 예시 주변 문구:

- "예시를 눌러 시작해도 괜찮습니다."
- "예시는 저장되지 않습니다. 고른 뒤 마음대로 바꿀 수 있습니다."
- "비슷한 문장이 있다면 눌러서 시작해보세요."

빈 입력 안내:

- "한 줄만 남겨도 충분합니다. 막혔던 순간을 적어주세요."
- "아직 적힌 내용이 없습니다. 짧은 문장으로 시작해도 괜찮습니다."

## 5. 마찰 지도에서 쓸 문구

마찰 지도는 처음 보면 표처럼 느껴질 수 있습니다. "어떻게 읽는지"를 화면 안에서 짧게 알려야 합니다.

권장 문구:

- "마찰 지도"
- "반복해서 막히는 위치를 생활 영역과 행동 단계로 나눠 보는 화면입니다."
- "먼저 자주 나타난 위치를 살펴보세요."
- "숫자가 높을수록 같은 위치에서 더 자주 나타났다는 뜻입니다."
- "아직 판단하지 않아도 괜찮습니다. 기록이 쌓이는 위치만 가볍게 봅니다."
- "가로로 밀어 더 볼 수 있습니다."

Top 3 문구:

- "자주 나타난 위치 Top 3"
- "먼저 살펴볼 만한 생활 영역과 막힌 지점입니다."
- "3회 나타난 위치입니다."

범례 문구:

- "0: 기록 없음"
- "1: 가끔 나타남"
- "2~3: 자주 나타남"
- "4 이상: 반복 신호"

피해야 할 문구:

- "문제 영역"
- "위험 신호"
- "나쁜 습관"
- "개선해야 할 부분"

## 6. 작은 실험 카드에서 쓸 문구

작은 실험 카드는 숙제나 목표 관리처럼 보이면 안 됩니다. "바로 고치기"가 아니라 "작게 바꿔보고 관찰하기"로 설명합니다.

기능명:

- "작은 실험 카드"는 유지합니다.
- 첫 안내에서는 "작게 바꿔보기"를 함께 사용합니다.

권장 문구:

- "바로 해결하려 하지 않아도 괜찮습니다."
- "반복되는 막힘 하나를 골라, 3~7일 동안 가볍게 관찰할 조건을 정해보세요."
- "왜 그랬을까요?"
- "이번에는 무엇을 작게 바꿔볼까요?"
- "무엇을 보면 충분할까요?"
- "예상대로 안 됐다면, 어떤 조건을 더 작게 바꿔볼 수 있을까요?"

단계별 문구:

| 현재 단계 | 사용자 친화 문구 |
| --- | --- |
| 어떤 마찰을 볼까요? | 어떤 막힘을 작게 바꿔볼까요? |
| 왜 막혔다고 생각하나요? | 왜 그랬을까요? |
| 어떤 작은 실험을 해볼까요? | 이번에는 무엇을 작게 바꿔볼까요? |
| 어떻게 관찰할까요? | 무엇을 보면 충분할까요? |

상태 문구:

- "진행 전": 아직 시작하지 않은 카드
- "진행 중": 지금 가볍게 확인 중
- "관찰 완료": 돌아볼 준비가 된 카드
- "보류": 나중에 다시 볼 카드

## 7. 피해야 할 표현

사용자를 평가하거나 진단하는 표현은 쓰지 않습니다.

피해야 할 표현:

- "실패했습니다"
- "의지 부족입니다"
- "문제 있는 습관입니다"
- "나쁜 습관입니다"
- "개선해야 합니다"
- "고쳐야 합니다"
- "위험합니다"
- "정상/비정상"
- "성공률"
- "달성률"
- "점수"
- "게으름"
- "자기관리 실패"
- "왜 못 했나요?"

대체 표현:

| 피할 표현 | 대체 표현 |
| --- | --- |
| 실패했습니다 | 여기서 막혔어요 |
| 의지 부족입니다 | 조건이 아직 맞지 않았을 수 있어요 |
| 문제 영역 | 자주 나타난 위치 |
| 고쳐야 합니다 | 조금 다르게 해볼 수 있습니다 |
| 왜 못 했나요? | 무엇이 어렵게 만들었나요? |
| 성공 기준 | 관찰 기준 |
| 달성률 | 지금 확인 중인 상태 |

## 8. 권장하는 마이크로카피 예시

첫 사용:

- "처음이라면 예시를 눌러 시작해도 괜찮습니다."
- "예시는 실제 기록으로 저장되지 않습니다."
- "한 줄만 남기면 지도에 위치가 표시됩니다."

저장 직후:

- "마찰이 지도에 표시됐어요. 기록이 쌓이면 반복되는 위치가 더 선명해집니다."
- "첫 막힘이 지도에 올라갔습니다. 아직 판단하지 말고 위치만 가볍게 봐도 충분합니다."

기록이 적을 때:

- "아직 패턴을 정하지 않아도 괜찮습니다. 몇 개의 기록이 더 쌓이면 흐름이 보이기 시작합니다."
- "지금은 관찰을 시작하는 단계입니다."

기록이 쌓였을 때:

- "자주 나타난 위치가 조금씩 보이기 시작합니다."
- "반복되는 막힘 하나를 골라 작게 바꿔볼 수 있습니다."

작은 실험:

- "정답을 찾는 것이 아니라, 부담을 줄일 조건을 가볍게 확인합니다."
- "3일, 5일, 7일 중 부담 없는 기간을 골라보세요."
- "예상대로 안 돼도 괜찮습니다. 조건을 더 작게 조정할 단서가 됩니다."

삭제:

- "삭제"
- "기록 삭제"
- "실험 삭제"

빈 상태:

- "아직 기록된 막힘이 없습니다."
- "거창한 기록이 아니어도 괜찮습니다. 오늘 막혔던 순간을 한 줄로 남겨보세요."
- "기록이 쌓이면 반복해서 막히는 위치가 여기에 나타납니다."

## 적용 원칙

- 첫 화면과 입력 폼에서는 "막힘"을 우선 사용합니다.
- 기능명과 지도 이름에는 "마찰"을 유지합니다.
- 분석처럼 보이는 문구는 "힌트", "가능성", "관찰"로 낮춥니다.
- 사용자가 해야 할 일을 늘리는 말보다, 막힘을 덜어내는 말을 사용합니다.
- 모든 문구는 "더 잘해야 한다"가 아니라 "어디서 막히는지 같이 보자"는 방향을 유지합니다.
