# 마찰지도 v0.2 테스트 체크리스트

이 문서는 v0.2 작은 실험 카드 기능이 기존 v0.1 흐름을 해치지 않고 동작하는지 확인하기 위한 체크리스트입니다.

## 1. 빌드 테스트

명령:

```bash
pnpm build
```

Windows PowerShell:

```powershell
pnpm.cmd build
```

합격 기준:

- 빌드가 오류 없이 성공합니다.
- TypeScript 오류가 없습니다.
- `window is not defined`, `localStorage is not defined` 오류가 없습니다.

## 2. 마찰 기록 유지 테스트

아래 기록을 입력합니다.

| 항목 | 값 |
| --- | --- |
| 내용 | 운동하려고 했는데 또 미뤘다 |
| 감정 | 부담 |
| 생활 영역 | 건강 |
| 마찰 단계 | 시작 전 마찰 |
| 강도 | 4 |

합격 기준:

- 기록 카드가 표시됩니다.
- 새로고침 후에도 기록이 유지됩니다.
- 카드에 내용, 감정, 생활 영역, 마찰 단계, 강도, 작성 시간이 보입니다.

## 3. 마찰 지도와 반복 요약 테스트

합격 기준:

- `건강 × 시작 전 마찰` 개수가 1 증가합니다.
- 반복 요약에 기록된 생활 영역, 마찰 단계, 감정이 반영됩니다.
- 기존 기록이 있다면 전체 저장 데이터 기준으로 계산됩니다.

## 4. 작은 실험 생성 테스트

아래 실험 카드를 작성합니다.

| 항목 | 값 |
| --- | --- |
| 실험 이름 | 운동 시작 마찰 줄이기 |
| 원인 가설 | 운동 전체를 해야 한다고 생각해서 시작이 커진 것 같다 |
| 실험 기간 | 5일 |
| 작은 실험 | 5일 동안 운동을 목표로 하지 않고 운동복만 입어본다 |
| 성공 기준 | 실제 운동 여부가 아니라 운동복을 입었는지만 본다 |
| 예상대로 안 됐을 때의 해석 | 실험 단위가 아직 컸다는 신호로 본다 |

합격 기준:

- 실험 카드가 생성됩니다.
- 연결된 마찰 기록 내용이 표시됩니다.
- 새 실험의 기본 상태가 `진행 전`입니다.
- 생성일과 수정일이 표시됩니다.

## 5. 공백 입력 방지 테스트

실험 작성 폼의 필수 입력값을 비우거나 공백만 넣고 저장합니다.

합격 기준:

- 빈 실험 카드가 생성되지 않습니다.
- 부드러운 안내 문구가 표시됩니다.
- 콘솔 오류가 없습니다.

## 6. 실험 기간 제한 테스트

합격 기준:

- 실험 기간은 `3일`, `5일`, `7일` 중 하나만 선택할 수 있습니다.
- 저장된 `durationDays` 값은 `3`, `5`, `7` 중 하나입니다.

## 7. 상태 변경 테스트

상태를 아래 순서로 변경합니다.

```text
진행 전 -> 진행 중 -> 관찰 완료 -> 보류
```

합격 기준:

- 화면의 현재 상태가 선택한 값으로 바뀝니다.
- `updatedAt`이 갱신됩니다.
- 새로고침 후에도 마지막 상태가 유지됩니다.
- 상태 변경은 점수나 판정처럼 표현되지 않습니다.

## 8. 새로고침 유지 테스트

실험 카드를 만든 뒤 브라우저를 새로고침합니다.

합격 기준:

- 마찰 기록이 유지됩니다.
- 실험 카드가 유지됩니다.
- 실험 상태가 유지됩니다.
- 연결된 마찰 기록 내용이 표시됩니다.

## 9. 삭제 테스트

실험 카드의 삭제 버튼을 누릅니다.

합격 기준:

- 해당 실험 카드가 목록에서 사라집니다.
- 새로고침 후에도 삭제된 실험 카드가 다시 나타나지 않습니다.
- 마찰 기록은 그대로 유지됩니다.

## 10. 연결된 마찰 기록 삭제 시 안정성 테스트

실험 카드와 연결된 마찰 기록을 삭제합니다.

합격 기준:

- 실험 카드 자체는 유지됩니다.
- 앱이 깨지지 않습니다.
- 아래 문구가 표시됩니다.

```text
연결된 마찰 기록을 찾을 수 없습니다. 실험 카드는 그대로 유지됩니다.
```

## 11. 다크모드 테스트

합격 기준:

- 다크모드 토글 버튼이 동작합니다.
- 새로고침 후에도 선택한 테마가 유지됩니다.
- 작은 실험 카드 폼과 목록의 텍스트 대비가 너무 낮지 않습니다.

## 12. 모바일 QA 체크 항목

브라우저 개발자도구의 디바이스 모드에서 확인합니다.

권장 크기:

- iPhone SE
- Pixel 7
- iPad

체크 항목:

| 항목 | 기준 |
| --- | --- |
| 입력 폼 | 화면 밖으로 튀지 않음 |
| 실험 기간 라디오 | 손가락으로 누르기 쉬움 |
| 실험 카드 | 제목과 상태 컨트롤이 겹치지 않음 |
| 마찰 지도 | 가로 스크롤 가능 |
| 기록 카드 | 배지와 삭제 버튼이 겹치지 않음 |
| 다크모드 | 글자가 잘 보임 |

## 13. UX 문구 점검

합격 기준:

- 사용자를 비난하는 표현이 없습니다.
- 사용자를 진단하는 표현이 없습니다.
- “가설”, “실험”, “관찰”, “패턴”, “마찰” 중심의 언어를 사용합니다.
- “성공률”, “점수”, “달성률”, “랭킹”, “streak” 같은 표현을 기능으로 사용하지 않습니다.

## 14. 콘솔 확인

아래 오류가 없어야 합니다.

```text
Hydration failed
localStorage is not defined
Cannot read properties of undefined
Warning: Each child in a list should have a unique "key" prop
```

## v0.2 합격 판정

필수 통과 항목:

- `pnpm build` 성공
- 마찰 기록 추가와 새로고침 유지
- 작은 실험 카드 생성
- 상태 변경과 새로고침 유지
- 삭제 후 `localStorage` 반영
- 연결된 마찰 기록이 없어도 실험 카드가 깨지지 않음

위 항목이 모두 통과하면 v0.2는 합격으로 봅니다.
