# 마찰지도 v0.2.0 릴리스 노트

## 버전

v0.2.0

## 릴리스 요약

마찰지도 v0.2.0은 v0.1의 “마찰을 기록하고 지도화하는 흐름” 위에, 기록된 마찰을 사용자가 직접 설계한 작은 실험 카드로 바꾸는 기능을 더한 버전입니다.

이번 버전의 핵심 흐름은 다음과 같습니다.

```text
마찰 기록 -> 반복 위치 확인 -> 작은 실험 카드 작성 -> 상태 변경 -> 새로고침 후 유지
```

작은 실험 카드는 정답이나 조언을 제공하지 않습니다. 사용자가 직접 원인 가설과 실험을 적고, 3일, 5일, 7일 동안 가볍게 관찰할 수 있게 돕습니다.

## 새로 추가된 기능

- 작은 실험 카드 작성 폼
- 기존 마찰 기록 선택
- 실험 이름 입력
- 원인 가설 입력
- 실험 기간 `3일`, `5일`, `7일` 선택
- 작은 실험 입력
- 성공 기준 입력
- 예상대로 안 됐을 때의 해석 입력
- 작은 실험 카드 목록
- 실험 상태 변경
- 실험 삭제
- 실험 `createdAt`, `updatedAt` 저장
- 기존 프리뷰 데이터에 `status`, `updatedAt`이 없을 때 기본값을 보완하는 로드 처리
- 연결된 마찰 기록이 삭제되어도 실험 카드가 깨지지 않도록 대체 문구 표시

## 유지된 v0.1 기능

- 마찰 기록 추가
- 빈 입력과 공백 입력 저장 방지
- 마찰 기록 `localStorage` 저장
- 새로고침 후 기록 유지
- 기록 카드 최신순 표시
- 기록 삭제와 삭제 상태 저장
- 생활 영역 × 마찰 단계 마찰 지도
- 반복 마찰 요약 Top 3
- 최근 7일 기준 주간 리포트
- 다크모드 토글과 테마 저장
- 모바일 한 열 레이아웃과 마찰 지도 가로 스크롤

## 수정/정리된 구조

- `FrictionExperimentStatus` 타입 추가
- `FrictionExperiment.durationDays`를 `3 | 5 | 7`로 제한
- `FrictionExperiment`에 `status`, `updatedAt` 추가
- `createExperiment()`가 저장까지 하지 않고 새 실험 객체만 생성하도록 정리
- `loadExperiments()`에서 잘못된 데이터는 안전하게 거르고, 보완 가능한 기존 데이터는 기본값으로 마이그레이션
- `ExperimentBuilder`를 v0.2 정식 입력 폼으로 정리
- `ExperimentList`에 상태 변경 UI와 날짜 표시 추가
- `ExperimentStatusControl` 컴포넌트 추가
- `page.tsx`에 실험 생성, 상태 변경, 삭제, 저장 흐름 연결

## 알려진 한계

- 데이터는 현재 브라우저의 `localStorage`에만 저장됩니다.
- 서버 저장, 로그인, 기기 간 동기화는 없습니다.
- 작은 실험 카드는 수정 기능을 아직 제공하지 않습니다.
- 실험 회고 메모는 아직 없습니다.
- 데이터 내보내기와 가져오기는 아직 없습니다.
- 모바일 기본 구조는 유지되지만 실제 기기별 세부 QA가 더 필요합니다.
- 이전 테스트 데이터가 남아 있으면 요약과 지도 개수는 해당 데이터까지 포함해 계산됩니다.
