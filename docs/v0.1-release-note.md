# 마찰지도 v0.1.0 릴리스 노트

## 버전

v0.1.0

## 릴리스 요약

마찰지도 v0.1.0은 "하려고 했지만 이상하게 막혔던 순간"을 한 줄로 기록하고, 그 기록을 생활 영역, 감정, 마찰 단계, 강도로 분류한 뒤 반복되는 위치를 확인할 수 있게 만든 첫 번째 MVP입니다.

이번 버전의 핵심 기준은 다음 흐름입니다.

```text
한 줄 기록 -> 저장 -> 새로고침 후 유지 -> 카드 표시 -> 마찰 지도 개수 반영 -> 삭제 시 함께 반영
```

v0.1.0은 위 흐름과 `pnpm build`를 통과한 상태를 기준점으로 삼습니다.

## 구현된 기능

- 마찰 기록 입력
- 감정, 생활 영역, 마찰 단계, 강도 선택
- 빈 입력과 공백 입력 저장 방지
- 마찰 기록 `localStorage` 저장
- 새로고침 후 기록 유지
- 기록 카드 최신순 표시
- 기록 삭제와 삭제 상태 저장
- 생활 영역 x 마찰 단계 마찰 지도
- 반복 마찰 요약 Top 3
- 최근 7일 기준 주간 리포트
- 다크모드 토글과 테마 저장
- 모바일 한 열 레이아웃과 마찰 지도 가로 스크롤

## 수정된 문제

- 강도 선택이 range 슬라이더일 때 입력 안정성이 떨어질 수 있어, 1~5 라디오 선택 UI로 변경했습니다.
- 브라우저 환경에서만 `localStorage`에 접근하도록 저장 유틸을 구성했습니다.
- 기록 삭제 시 화면 상태와 `localStorage`가 함께 갱신되도록 확인했습니다.
- 지도와 요약이 동일한 기록 상태를 기준으로 계산되도록 연결했습니다.

## 알려진 한계

- 데이터는 현재 브라우저의 `localStorage`에만 저장됩니다.
- 서버 저장, 로그인, 기기 간 동기화는 없습니다.
- 작은 실험 카드 관련 타입, 저장 유틸, 컴포넌트는 v0.2 프리뷰 구조로 코드에 남아 있지만 v0.1 화면에는 노출하지 않습니다.
- 마찰 지도는 모바일에서 가로 스크롤이 가능하지만, 작은 화면의 실제 사용감은 추가 QA가 필요합니다.
- 동률 요약 항목의 순서는 중요 기준이 아니며, 개수 정확성을 우선합니다.
- 데이터 내보내기와 가져오기는 아직 없습니다.

## 다음 단계 추천

- Git 커밋과 `v0.1.0` 태그로 기준점을 고정합니다.
- 실제 모바일 기기 또는 브라우저 디바이스 모드에서 화면 QA를 진행합니다.
- 작은 실험 카드를 v0.2에서 정식화할지 설계하고, 진행 상태, 관찰 메모, 회고 흐름의 범위를 정합니다.
- 기록 검색, 필터, 로컬 백업 같은 관리 기능을 검토합니다.
