# 마찰지도 v0.3.3 사용자 테스트 스크립트

## 1. 테스트 목적

마찰지도 v0.3.2 UI와 v0.3.3 위험 검토 이후, 실제 사용자가 앱을 처음 열었을 때 아래 흐름을 자연스럽게 이해하고 실행하는지 확인한다.

```text
남기기 -> 지도 -> 시도 -> 돌아보기
```

이번 테스트는 기능 검증보다 사용성 검증이 목적이다. 앱이 치료나 진단 도구처럼 보이는지, 입력이 부담스럽지는 않은지, 지도와 시도 카드가 실제로 이해되는지를 본다.

## 2. 권장 테스트 규모

- 최소 5명
- 가능하면 8~12명
- Android 실기기 우선
- 테스트 시간: 1명당 15~20분
- 테스트 방식: 옆에서 관찰하거나 화면 공유로 진행

## 3. 참여자 유형

가능하면 아래 유형이 섞이게 모집한다.

- 자기관리 앱을 자주 쓰는 사람
- 자기관리 앱을 거의 안 쓰는 사람
- 바쁜 직장인 또는 학생
- 개인정보에 민감한 사람
- 스마트폰 앱 사용이 익숙하지 않은 사람

## 4. 테스트 전 준비

테스트 시작 전 아래를 준비한다.

- Android 테스트 빌드 설치
- 기존 앱 데이터 삭제 또는 앱 재설치
- 네트워크가 끊겨도 앱이 열리는지 확인
- 기록 내보내기 파일이 저장될 위치 확인
- 관찰 기록용 메모지 또는 스프레드시트 준비

참여자에게는 앱의 정답이나 사용법을 자세히 설명하지 않는다. 처음 보는 사람이 자연스럽게 이해하는지 보기 위해서다.

## 5. 시작 안내 문구

테스터에게 아래처럼 안내한다.

```text
이 앱은 자꾸 막히는 순간을 한 줄로 남기고,
반복되는 위치를 보며 다음에 덜 버겁게 해볼 방법을 정하는 개인 기록 앱입니다.

정답을 맞히는 테스트가 아닙니다.
헷갈리는 부분이 있으면 그대로 말해주세요.
```

## 6. 핵심 과제

### 과제 1. 첫 인상 확인

앱을 처음 열고 10초 동안 보게 한 뒤 묻는다.

- 이 앱이 무엇을 하는 앱처럼 보이나요?
- 가장 먼저 눌러보고 싶은 곳은 어디인가요?
- 치료나 진단 앱처럼 느껴지나요, 개인 기록 앱처럼 느껴지나요?

합격 기준:
- “막힌 순간을 기록하는 앱”이라는 의미를 대략 이해한다.
- 첫 행동이 “기록하기”라는 점을 찾는다.

### 과제 2. 45초 안에 첫 기록 남기기

참여자에게 말한다.

```text
최근에 하려고 했지만 잘 안 됐던 순간을 하나 떠올려서 기록해보세요.
개인적인 내용을 적기 부담스러우면 예시 칩을 사용해도 됩니다.
```

관찰할 것:
- 입력창을 찾는 데 걸린 시간
- 예시 칩을 이해하는지
- 칩 선택 후 값이 자동으로 채워지는 것을 알아차리는지
- 감정/영역/단계/버거움 선택이 부담스럽지는 않은지
- 저장 후 피드백 메시지를 보는지

합격 기준:
- 45초 안에 첫 기록을 저장한다.
- 빈 입력 오류가 위협적으로 느껴지지 않는다.
- 저장 후 “지도에 표시됐다”는 의미를 이해한다.

### 과제 3. 지도 화면 이해

기록을 2~3개 추가한 뒤 지도 탭으로 이동하게 한다.

확인할 것:
- “자주 막힌 위치” Top 3를 먼저 읽는지
- 생활 영역 x 막힌 순간 구조를 이해하는지
- 가로 스크롤 안내를 이해하는지
- 범례가 도움이 되는지

질문:
- 이 화면에서 무엇을 알 수 있다고 느끼나요?
- 숫자가 커질수록 어떤 의미라고 이해했나요?
- “문제 영역”처럼 평가받는 느낌이 있나요?

합격 기준:
- 지도는 “내가 어디서 자주 막히는지 보는 화면”으로 이해된다.
- Top 3가 전체 표보다 먼저 도움이 된다.

### 과제 4. 시도 카드 만들기

참여자에게 말한다.

```text
방금 남긴 기록 중 하나를 골라,
다음에 조금 덜 버겁게 해볼 방법을 하나 만들어보세요.
```

관찰할 것:
- 기록 선택이 쉬운지
- “왜 막혔을까요?” 질문이 부담스럽지 않은지
- “다음에 작게 바꿔볼 것”을 적을 수 있는지
- “어떻게 확인할까요?”가 성과 평가처럼 느껴지지 않는지
- 상태 변경을 이해하는지

합격 기준:
- 시도 카드가 숙제처럼 느껴지지 않는다.
- 사용자가 직접 작은 방법을 적을 수 있다.
- 상태 변경이 점수나 성공 판정처럼 보이지 않는다.

### 과제 5. 돌아보기와 기록 관리

돌아보기 탭에서 아래를 시도하게 한다.

- 전체 기록 확인
- 기록 내보내기 위치 찾기
- 개인정보 안내 읽기
- 모든 기록 삭제 버튼 위치 확인

실제로 전체 삭제는 테스트 마지막에만 수행한다.

질문:
- 기록이 기기 안에 저장된다는 설명이 신뢰감을 주나요?
- 내보내기와 삭제 기능을 찾기 쉬웠나요?
- 삭제 버튼이 너무 무섭거나 너무 약하게 보이나요?

합격 기준:
- 사용자는 자신의 기록을 통제할 수 있다고 느낀다.
- 개인정보 안내가 과장되거나 불명확하게 느껴지지 않는다.

## 7. 관찰 기록 양식

각 참여자마다 아래 항목을 기록한다.

```text
참여자 번호:
기기:
Android 버전:
글자 크기 설정:

첫 인상:
첫 기록까지 걸린 시간:
예시 칩 사용 여부:
지도 이해 여부:
시도 카드 작성 여부:
개인정보 신뢰도:

가장 헷갈린 단어:
가장 막힌 화면:
가장 좋았던 점:
다시 쓸 의향:
기타 메모:
```

## 8. 점수 기준

각 항목을 1~5점으로 기록한다.

| 항목 | 1점 | 3점 | 5점 |
| --- | --- | --- | --- |
| 첫 화면 이해 | 거의 이해 못함 | 대략 이해 | 바로 이해 |
| 첫 기록 쉬움 | 도움 없이는 어려움 | 조금 망설임 | 바로 저장 |
| 지도 이해 | 표처럼만 보임 | 일부 이해 | Top 3와 지도 의미 이해 |
| 시도 카드 부담 | 숙제처럼 느껴짐 | 보통 | 가볍게 느껴짐 |
| 개인정보 신뢰 | 불안함 | 보통 | 안심됨 |
| 재사용 의향 | 없음 | 가끔 | 다시 써보고 싶음 |

## 9. 성공 기준

v0.3.3 사용자 테스트 통과 기준은 아래와 같다.

- 5명 중 4명 이상이 첫 화면 목적을 이해한다.
- 5명 중 4명 이상이 45초 안에 첫 기록을 저장한다.
- 5명 중 3명 이상이 지도 Top 3의 의미를 설명할 수 있다.
- 5명 중 3명 이상이 시도 카드를 만들 수 있다.
- 5명 중 4명 이상이 앱이 치료/진단 도구처럼 느껴지지 않는다고 답한다.
- 개인정보 안내를 보고 5명 중 4명 이상이 기록 저장 방식을 이해한다.

## 10. 실패 판정이 아니라 조정 신호

테스트에서 사용자가 막히는 것은 앱을 탓하거나 사용자를 탓하는 근거가 아니다. 어디에서 설명이 부족했는지, 어떤 표현이 딱딱했는지, 어떤 화면이 너무 길었는지 찾는 재료로 본다.
