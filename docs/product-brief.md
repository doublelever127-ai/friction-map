# 마찰지도 제품 정의 문서

## 제품 한 문장 정의

마찰지도는 사용자가 하려고 했지만 이상하게 막혔던 순간을 기록하고, 그 마찰을 생활 영역, 행동 단계, 감정, 원인 가설로 분해해 작은 해결 실험으로 바꿔주는 자기이해 도구입니다.

> 할 일을 더 늘리지 않습니다. 당신을 막는 마찰을 발견합니다.

## 이 앱이 해결하려는 문제

많은 사용자는 무언가를 하지 못했을 때 그 이유를 의지 부족, 게으름, 자기관리 실패로 해석합니다. 하지만 실제로는 너무 큰 시작점, 애매한 다음 행동, 피로, 환경의 방해, 감정적 부담, 보이지 않는 저항감처럼 작은 마찰들이 행동을 멈추게 만드는 경우가 많습니다.

마찰지도는 사용자가 실패를 더 세게 밀어붙이도록 돕지 않습니다. 대신 막혔던 순간을 부드럽게 들여다보고, 그 안에 숨어 있는 마찰을 발견하며, 다음번에는 조금 덜 막히도록 작은 실험을 설계하게 돕습니다.

## 이 앱이 기록하는 핵심 대상

이 앱이 기록하는 핵심 대상은 할 일, 습관, 감정 자체가 아니라 "하려고 했지만 이상하게 막혔던 순간"입니다.

예시는 다음과 같습니다.

- 책상에 앉았지만 문서를 열지 못한 순간
- 운동복까지 입었지만 집 밖으로 나가지 못한 순간
- 답장을 해야 한다고 생각했지만 계속 미룬 순간
- 잠들 준비를 하려 했지만 휴대폰을 내려놓지 못한 순간
- 정리하려고 했지만 어디서부터 시작해야 할지 몰라 멈춘 순간

기록의 중심은 "무엇을 못 했는가"가 아니라 "어디에서, 어떤 느낌으로, 어떤 조건 때문에 멈췄는가"입니다.

## 기존 할 일 앱/습관 앱/감정 일기와의 차이

### 할 일 앱과의 차이

할 일 앱은 해야 할 일을 목록화하고 완료 여부를 관리합니다. 마찰지도는 해야 할 일을 더 많이 만들지 않습니다. 이미 하려고 했지만 막힌 행동의 앞뒤 맥락을 분석해, 다음 시도에서 마찰을 줄이는 데 집중합니다.

### 습관 앱과의 차이

습관 앱은 반복 수행, streak, 달성률, 리마인더를 중심으로 설계되는 경우가 많습니다. 마찰지도는 연속 성공을 추적하지 않고, 실패나 중단을 벌점처럼 다루지 않습니다. 반복보다 이해를, 점수보다 관찰을 우선합니다.

### 감정 일기와의 차이

감정 일기는 하루의 감정 상태를 기록하고 회고하는 데 초점을 둡니다. 마찰지도는 감정을 중요한 단서로 보지만, 감정 자체를 최종 기록 대상으로 삼지는 않습니다. 감정은 행동이 막힌 이유를 추정하기 위한 한 조각의 정보입니다.

## 주요 사용자 경험

1. 사용자는 "오늘 무엇을 못 했나요?"가 아니라 "어디에서 막혔나요?"라는 질문을 만납니다.
2. 사용자는 막힌 순간을 짧게 적습니다.
3. 앱은 그 순간을 생활 영역, 행동 단계, 감정, 원인 가설로 나누어 정리하도록 돕습니다.
4. 모든 분석 결과는 확정적 진단이 아니라 "이런 가능성이 있어 보여요" 수준의 가설로 표현됩니다.
5. 사용자는 큰 목표를 추가하는 대신 다음 시도에서 해볼 수 있는 작은 해결 실험을 정합니다.
6. 시간이 지나면 사용자는 자신이 자주 막히는 영역, 단계, 감정, 환경 조건을 지도로 확인합니다.
7. 앱은 사용자를 평가하지 않고, 반복되는 마찰 패턴을 차분하게 보여줍니다.

## MVP 범위

MVP는 사용자가 막힌 순간을 기록하고, 그 기록을 구조화하며, 작은 해결 실험으로 전환할 수 있는 최소 흐름에 집중합니다.

- 마찰 기록 작성
- 생활 영역 선택
- 행동 단계 선택
- 감정 선택 또는 직접 입력
- 원인 가설 선택 또는 직접 입력
- 작은 해결 실험 작성
- 기록 목록 보기
- 기록 상세 보기
- 반복되는 마찰 패턴의 간단한 요약 보기
- 부드럽고 비판 없는 문구 체계

초기 MVP에서는 자동 분석보다 사용자가 스스로 분해할 수 있도록 돕는 입력 구조와 언어가 더 중요합니다.

## 일부러 넣지 않을 기능

마찰지도는 사용자를 더 강하게 몰아붙이는 기능을 의도적으로 넣지 않습니다.

- streak
- 점수, 레벨, 랭킹
- 경쟁 요소
- SNS 공유
- 공개 피드
- 과한 알림
- 실패 경고
- 죄책감을 유발하는 문구
- 자동으로 목표를 늘리는 기능
- 생산성 점수화
- 감정이나 성향에 대한 단정적 진단

이 앱은 사용자를 더 빽빽하게 관리하지 않고, 막힘을 더 잘 이해하도록 돕습니다.

## 데이터 모델 초안

### FrictionEntry

- id: 기록 고유 ID
- title: 막힌 순간을 요약한 짧은 문장
- description: 사용자가 자유롭게 적은 상황 설명
- lifeArea: 생활 영역
- actionStage: 막힌 행동 단계
- emotions: 사용자가 느낀 감정 목록
- hypothesis: 원인 가설
- experiment: 다음에 시도할 작은 해결 실험
- createdAt: 기록 생성 시각
- updatedAt: 기록 수정 시각

### LifeArea

- work: 일과 공부
- body: 몸과 건강
- home: 집과 생활
- relationship: 관계와 소통
- money: 돈과 관리
- rest: 휴식과 회복
- self: 자기이해와 돌봄
- other: 기타

### ActionStage

- beforeStart: 시작 전
- starting: 시작하는 순간
- continuing: 이어가는 중
- switching: 다른 행동으로 전환하는 순간
- finishing: 마무리하는 순간
- afterFinish: 끝낸 뒤 정리하는 순간

### Emotion

- anxious: 불안
- tired: 피곤함
- overwhelmed: 압도됨
- bored: 지루함
- guilty: 죄책감
- resistant: 저항감
- blank: 멍함
- unsure: 잘 모르겠음
- other: 직접 입력

### Hypothesis

- tooBig: 시작점이 너무 큼
- unclearNextStep: 다음 행동이 애매함
- lowEnergy: 에너지가 부족함
- emotionalLoad: 감정적 부담이 큼
- environmentFriction: 환경이 방해함
- timingMismatch: 시간대나 리듬이 맞지 않음
- fearOfResult: 결과에 대한 부담이 있음
- hiddenNeed: 다른 욕구가 더 강하게 작동함
- unknown: 아직 잘 모르겠음
- custom: 직접 입력

## 화면 구성 초안

### 1. 홈

- 오늘 또는 최근에 막혔던 순간을 기록하는 진입점
- 최근 마찰 기록 목록
- 반복해서 나타나는 마찰 패턴의 짧은 요약
- "새 마찰 기록" 버튼

### 2. 새 마찰 기록

- 막힌 순간 한 줄 입력
- 상황 설명 입력
- 생활 영역 선택
- 행동 단계 선택
- 감정 선택
- 원인 가설 선택
- 작은 해결 실험 입력

### 3. 기록 상세

- 사용자가 적은 막힌 순간
- 분해된 마찰 정보
- 원인 가설
- 작은 해결 실험
- 나중에 다시 볼 수 있는 메모

### 4. 마찰 지도

- 생활 영역별 마찰 분포
- 행동 단계별 마찰 분포
- 자주 함께 나타나는 감정과 원인 가설
- 반복되는 패턴에 대한 부드러운 요약

### 5. 설정

- 데이터 관리
- 알림 사용 여부
- 언어 톤 관련 기본 설정
- 개인정보와 로컬 저장 관련 안내

## 말투와 UX 원칙

마찰지도의 언어는 차분하고 부드러워야 합니다. 사용자를 평가하거나 교정하려는 말투가 아니라, 함께 관찰하고 정리하는 말투를 사용합니다.

### 말투 원칙

- "실패했습니다"보다 "여기서 막혔어요"라고 표현합니다.
- "원인입니다"보다 "이런 가능성이 있어 보여요"라고 표현합니다.
- "다시 도전하세요"보다 "다음에는 조금 다르게 해볼 수 있어요"라고 표현합니다.
- "왜 못 했나요?"보다 "무엇이 어렵게 만들었나요?"라고 묻습니다.
- "목표를 추가하세요"보다 "마찰을 줄이는 작은 실험을 정해볼까요?"라고 제안합니다.

### UX 원칙

- 기록은 짧게 시작할 수 있어야 합니다.
- 선택지는 사용자를 몰아붙이지 않아야 합니다.
- "잘 모르겠음"을 항상 유효한 선택지로 제공합니다.
- 분석 결과는 확정이 아니라 가설로 보여줍니다.
- 빈 화면은 성취 압박이 아니라 관찰 초대로 구성합니다.
- 알림은 최소화하고, 사용자가 원할 때만 켤 수 있게 합니다.
- 모든 화면은 "더 잘해야 한다"가 아니라 "덜 막히게 해보자"는 방향을 유지합니다.
