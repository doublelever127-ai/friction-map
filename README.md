# 마찰지도

마찰지도는 사용자가 하려고 했지만 이상하게 막혔던 순간을 기록하고, 반복되는 마찰의 위치를 발견한 뒤 작은 실험으로 바꿔보는 자기이해 도구입니다.

> 할 일을 더 늘리지 않습니다. 당신을 막는 마찰을 발견합니다.

## 핵심 철학

- 실패를 비난하지 않습니다.
- 의지 부족으로 해석하지 않습니다.
- 분석은 진단이 아니라 가설로 다룹니다.
- 목표를 더 늘리는 대신 마찰을 줄이는 데 집중합니다.
- streak, 점수 경쟁, 과한 알림, SNS 공유를 넣지 않습니다.
- 앱의 언어는 차분하고 부드럽게 유지합니다.

## v0.2 주요 기능

- 마찰 기록 추가: 막혔던 순간, 감정, 생활 영역, 마찰 단계, 강도를 기록합니다.
- 마찰 기록 목록: 저장된 기록을 최신순 카드로 확인하고 삭제할 수 있습니다.
- 마찰 지도: 생활 영역과 마찰 단계 조합별 기록 수를 히트맵으로 보여줍니다.
- 반복 마찰 요약: 자주 나타난 생활 영역, 마찰 단계, 감정을 Top 3로 정리합니다.
- 주간 리포트: 최근 7일 기록을 기준으로 관찰해볼 패턴을 정리합니다.
- 작은 실험 카드: 특정 마찰 기록을 골라 원인 가설, 작은 실험, 성공 기준, 예상대로 안 됐을 때의 해석을 직접 작성합니다.
- 실험 상태 관리: 작은 실험의 현재 관찰 위치를 `진행 전`, `진행 중`, `관찰 완료`, `보류` 중 하나로 표시합니다.
- 다크모드: 외부 라이브러리 없이 `dark` 클래스를 토글하고 테마를 저장합니다.

## 작은 실험 카드

작은 실험 카드는 앱이 조언을 생성하는 기능이 아닙니다. 사용자가 직접 마찰 기록 하나를 선택하고, 왜 막혔는지에 대한 가설과 3일, 5일, 7일 동안 가볍게 관찰할 실험을 작성하는 구조입니다.

실험 카드에는 아래 정보가 저장됩니다.

- 연결된 마찰 기록
- 실험 이름
- 원인 가설
- 작은 실험
- 실험 기간
- 성공 기준
- 예상대로 안 됐을 때의 해석
- 현재 상태
- 생성일과 수정일

연결된 마찰 기록이 삭제되어도 실험 카드는 유지되며, 연결 기록을 찾을 수 없다는 부드러운 안내 문구를 표시합니다.

## 실험 상태

실험 상태는 점수나 성공/실패 판정이 아니라 현재 관찰 위치를 표시하기 위한 값입니다.

| 상태 | 의미 |
| --- | --- |
| 진행 전 | 아직 시작하지 않은 실험 |
| 진행 중 | 지금 관찰 중인 실험 |
| 관찰 완료 | 돌아볼 준비가 된 실험 |
| 보류 | 잠시 멈춰둔 실험 |

## 기술 스택

- Next.js App Router
- TypeScript
- Tailwind CSS
- `localStorage` 기반 로컬 저장

현재 버전에서는 서버, 데이터베이스, 로그인, AI API를 사용하지 않습니다.

## localStorage 저장 키

| 데이터 | 저장 키 |
| --- | --- |
| 마찰 기록 | `friction-map.logs.v1` |
| 작은 실험 카드 | `friction-map.experiments.v1` |
| 테마 설정 | `friction-map.theme` |

## 실행 방법

의존성을 설치합니다.

```bash
pnpm install
```

개발 서버를 실행합니다.

```bash
pnpm dev
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:3000
```

Windows PowerShell에서 `pnpm` 실행이 막히는 경우에는 다음처럼 실행할 수 있습니다.

```powershell
pnpm.cmd dev
```

## 빌드 방법

```bash
pnpm build
```

Windows PowerShell에서는 다음 명령도 사용할 수 있습니다.

```powershell
pnpm.cmd build
```

## 제품 언어 원칙

마찰지도는 사용자를 평가하지 않습니다. 기록과 실험은 정답을 찾기 위한 판정이 아니라, 반복되는 패턴을 관찰하기 위한 힌트입니다.

앱에서는 “마찰”, “패턴”, “가설”, “관찰”, “작은 실험”이라는 언어를 중심으로 사용합니다.
