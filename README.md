# 마찰지도

마찰지도는 사용자가 하려고 했지만 이상하게 막혔던 순간을 기록하고, 반복되는 마찰의 위치를 발견하도록 돕는 자기이해 도구입니다. 작은 해결 실험으로 바꾸는 흐름은 v0.2에서 정식화할 예정입니다.

> 할 일을 더 늘리지 않습니다. 당신을 막는 마찰을 발견합니다.

## 핵심 철학

- 실패를 비난하지 않습니다.
- 의지 부족으로 해석하지 않습니다.
- 분석은 진단이 아니라 가설로 다룹니다.
- 목표를 더 늘리는 대신 마찰을 줄이는 데 집중합니다.
- streak, 점수 경쟁, 과한 알림, SNS 공유를 넣지 않습니다.
- 앱의 언어는 차분하고 부드럽게 유지합니다.

## 주요 기능

- 마찰 기록 추가: 막혔던 순간, 감정, 생활 영역, 마찰 단계, 강도를 기록합니다.
- 로컬 저장: 기록과 테마 설정을 브라우저 `localStorage`에 저장합니다.
- 마찰 기록 목록: 저장된 기록을 최신순 카드로 확인하고 삭제할 수 있습니다.
- 마찰 지도: 생활 영역과 마찰 단계 조합별 기록 수를 히트맵으로 보여줍니다.
- 반복 마찰 요약: 자주 나타난 생활 영역, 마찰 단계, 감정을 Top 3로 정리합니다.
- 주간 리포트: 최근 7일 기록을 기준으로 관찰해볼 패턴을 정리합니다.
- 다크모드: 외부 라이브러리 없이 `dark` 클래스를 토글하고 테마를 저장합니다.

참고: 작은 실험 카드 관련 타입, 저장 유틸, 컴포넌트는 v0.2 프리뷰 구조로 코드에 남아 있지만 v0.1 화면에는 정식 기능으로 노출하지 않습니다.

## v0.1 통과 기준

v0.1은 아래 핵심 흐름이 통과한 상태를 기준점으로 삼습니다.

```text
한 줄 기록 -> 저장 -> 새로고침 후 유지 -> 카드 표시 -> 마찰 지도 개수 반영 -> 삭제 시 함께 반영
```

확인된 항목은 다음과 같습니다.

- `pnpm build` 성공
- dev 서버 화면 렌더링 확인
- 마찰 기록 추가와 삭제
- 빈 입력과 공백 입력 저장 방지
- 새로고침 후 `localStorage` 기록 유지
- 삭제 후 새로고침해도 삭제 상태 유지
- 마찰 지도 개수 정확성 확인
- 반복 마찰 요약 개수 확인
- 다크모드 토글과 새로고침 후 테마 유지
- 브라우저 콘솔에 치명적인 hydration, `localStorage`, key 경고 없음

## 기술 스택

- Next.js App Router
- TypeScript
- Tailwind CSS
- `localStorage` 기반 로컬 저장

초기 버전에서는 서버, 데이터베이스, 로그인, AI API를 사용하지 않습니다.

## localStorage 저장 키

| 데이터 | 저장 키 |
| --- | --- |
| 마찰 기록 | `friction-map.logs.v1` |
| 작은 실험 카드(v0.2 프리뷰 구조, 화면 비노출) | `friction-map.experiments.v1` |
| 테마 설정 | `friction-map.theme` |

## 실행 방법

의존성을 설치합니다.

```bash
pnpm install
```

개발 서버를 실행합니다.

```bash
pnpm dev
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:3000
```

Windows PowerShell에서 `pnpm` 실행이 막히는 경우에는 다음처럼 실행할 수 있습니다.

```powershell
pnpm.cmd dev
```

## 빌드 방법

```bash
pnpm build
```

Windows PowerShell에서는 다음 명령도 사용할 수 있습니다.

```powershell
pnpm.cmd build
```

## 제품 언어 원칙

마찰지도는 사용자를 평가하지 않습니다. 기록은 정답을 찾기 위한 판정이 아니라, 반복되는 패턴을 관찰하기 위한 힌트입니다.

앱에서는 “마찰”, “패턴”, “가설”, “관찰”, “작은 실험”이라는 언어를 중심으로 사용합니다.
